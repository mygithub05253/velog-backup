<h1 id="스프링-웹-개발-기초">스프링 웹 개발 기초</h1>
<hr />
<p>스프링 부트에서 웹을 개발하는 방식은 크게 <strong>정적 컨텐츠</strong>, <strong>MVC와 템플릿 엔진</strong>, <strong>API</strong> 세 가지로 구분된다.</p>
<h2 id="1-정적-컨텐츠-static-content">1. 정적 컨텐츠 (Static Content)</h2>
<hr />
<p>서버에서 별도의 가공이나 프로그래밍 로직 없이 파일을 있는 그대로 웹 브라우저에 전달하는 방식이다.</p>
<h3 id="11-동작-원리">1.1 동작 원리</h3>
<hr />
<p>웹 브라우저가 요청을 보냈을 때 스프링 부트는 다음과 같은 순서로 리소스를 탐색한다.</p>
<ol>
<li><strong>컨트롤러 우선순위:</strong> 스프링 컨테이너에서 해당 요청(URL)을 처리할 컨트롤러(<code>@Controller</code>)가 있는지 먼저 찾는다.</li>
<li><strong>리소스 반환:</strong> 컨트롤러가 매핑되어 있지 않다면, <code>resources/static</code> 경로 하위에서 해당 파일을 찾아 반환한다.</li>
</ol>
<h3 id="12-예제-코드">1.2 예제 코드</h3>
<hr />
<ul>
<li><strong>파일 경로:</strong> <code>resources/static/hello-static.html</code></li>
<li><strong>HTML 작성:</strong></li>
</ul>
<pre><code class="language-html">&lt;!DOCTYPE HTML&gt;
&lt;html&gt;
&lt;head&gt;
    &lt;title&gt;static content&lt;/title&gt;
    &lt;meta http-equiv=&quot;Content-Type&quot; content=&quot;text/html; charset=UTF-8&quot; /&gt;
&lt;/head&gt;
&lt;body&gt;
    정적 컨텐츠 입니다.
&lt;/body&gt;
&lt;/html&gt;</code></pre>
<ul>
<li><strong>실행:</strong> <code>http://localhost:8080/hello-static.html</code> 접속 시 HTML 파일 내용이 그대로 렌더링 된다.</li>
</ul>
<h2 id="2-mvc와-템플릿-엔진-mvc--template-engine">2. MVC와 템플릿 엔진 (MVC &amp; Template Engine)</h2>
<hr />
<p>서버 사이드에서 HTML을 동적으로 생성(렌더링)하여 클라이언트에게 전달하는 방식이다. 과거에는 View에 로직을 포함시켰으나, 현재는 역할에 따라 Model, View, Controller로 분리하여 개발한다.</p>
<h3 id="21-구성-요소">2.1 구성 요소</h3>
<hr />
<ul>
<li><strong>Controller:</strong> 비즈니스 로직을 처리하고, 화면에 필요한 데이터를 Model에 담아 View로 넘겨준다.</li>
<li><strong>Model:</strong> 화면(View)에서 사용할 데이터를 담는 객체이다.</li>
<li><strong>View:</strong> 화면을 그리는 역할(UI)에 집중한다.</li>
</ul>
<h3 id="22-동작-원리">2.2 동작 원리</h3>
<hr />
<ol>
<li><strong>요청 수신:</strong> 웹 브라우저가 요청(<code>localhost:8080/hello-mvc</code>)을 보낸다.</li>
<li><strong>Controller 실행:</strong> 내장 톰켓 서버를 거쳐 스프링 컨테이너 내의 매핑된 Controller 메서드가 호출된다.</li>
<li><strong>반환 (Return):</strong> Controller는 리턴 값으로 논리적인 뷰 이름(<code>hello-template</code>)을 반환하고, <code>Model</code>에 데이터(<code>name: spring</code>)를 담는다.</li>
<li><strong>ViewResolver 동작:</strong> <code>viewResolver</code>가 리턴된 뷰 이름에 해당하는 템플릿 파일(<code>resources/templates/hello-template.html</code>)을 찾는다.</li>
<li><strong>렌더링:</strong> 템플릿 엔진(Thymeleaf)이 템플릿 파일과 Model 데이터를 결합하여 HTML로 변환한 후 클라이언트에 응답한다.</li>
</ol>
<h3 id="23-예제-코드">2.3 예제 코드</h3>
<hr />
<p><strong>Controller (<code>HelloController.java</code>)</strong></p>
<pre><code class="language-java">@Controller
public class HelloController {
    @GetMapping(&quot;hello-mvc&quot;)
    public String helloMvc(@RequestParam(&quot;name&quot;) String name, Model model) {
        model.addAttribute(&quot;name&quot;, name); // 파라미터로 받은 name을 모델에 담음
        return &quot;hello-template&quot;; // 템플릿 파일 이름 리턴
    }
}</code></pre>
<p><strong>View (<code>resources/templates/hello-template.html</code>)</strong></p>
<pre><code class="language-html">&lt;html xmlns:th=&quot;http://www.thymeleaf.org&quot;&gt;
&lt;body&gt;
    &lt;p th:text=&quot;'hello' + ${name}&quot;&gt;hello! empty&lt;/p&gt;
&lt;/body&gt;
&lt;/html&gt;</code></pre>
<h2 id="3-api-application-programming-interface">3. API (Application Programming Interface)</h2>
<hr />
<p>HTML 같은 뷰 템플릿을 거치지 않고, HTTP 응답 본문(Body)에 데이터를 직접 실어서 보내는 방식이다. 주로 JSON 데이터 포맷을 사용한다.</p>
<h3 id="31-핵심-애노테이션">3.1 핵심 애노테이션</h3>
<hr />
<ul>
<li><strong><code>@ResponseBody</code>:</strong> 뷰 리졸버(<code>viewResolver</code>)를 사용하지 않고, 반환된 데이터를 HTTP Body에 직접 입력하겠다는 의미이다.</li>
</ul>
<h3 id="32-동작-원리">3.2 동작 원리</h3>
<hr />
<ol>
<li><strong>요청 수신:</strong> 웹 브라우저가 요청(<code>localhost:8080/hello-api</code>)을 보낸다.</li>
<li><strong>Controller 실행:</strong> <code>@ResponseBody</code>가 적용된 경우, 스프링은 <code>viewResolver</code> 대신 <strong><code>HttpMessageConverter</code></strong>를 동작시킨다.</li>
<li><strong>데이터 변환 및 응답 (HttpMessageConverter):</strong><ul>
<li><strong>문자열 반환 시:</strong> <code>StringHttpMessageConverter</code>가 동작하여 문자열 그대로를 반환한다.</li>
<li><strong>객체 반환 시:</strong> <code>MappingJackson2HttpMessageConverter</code>가 기본으로 동작하여 객체를 <strong>JSON</strong> 포맷으로 변환해 반환한다.</li>
<li>참고: 클라이언트의 HTTP Accept 헤더와 서버의 반환 타입 정보를 조합해 적절한 컨버터가 선택됨</li>
</ul>
</li>
</ol>
<h3 id="33-예제-코드-객체-반환">3.3 예제 코드 (객체 반환)</h3>
<p><strong>Controller (<code>HelloController.java</code>)</strong></p>
<pre><code class="language-java">@Controller
public class HelloController {
    @GetMapping(&quot;hello-api&quot;)
    @ResponseBody // HTTP Body에 데이터 직접 반환
    public Hello helloApi(@RequestParam(&quot;name&quot;) String name) {
        Hello hello = new Hello();
        hello.setName(name);
        return hello; // 객체 반환 -&gt; JSON으로 변환됨
    }

    static class Hello {
        private String name;

        public String getName() {
            return name;
        }
        public void setName(String name) {
            this.name = name;
        }
    }
}</code></pre>
<p><strong>실행 결과 (<code>http://localhost:8080/hello-api?name=spring</code>)</strong></p>
<pre><code class="language-json">{
    &quot;name&quot;: &quot;spring&quot;
}</code></pre>
<h2 id="4-비즈니스-요구사항-및-설계-예시--회원-관리">4. 비즈니스 요구사항 및 설계 (예시 : 회원 관리)</h2>
<hr />
<h3 id="41-일반적인-웹-애플리케이션-계층-구조">4.1 일반적인 웹 애플리케이션 계층 구조</h3>
<hr />
<p>스프링 웹 애플리케이션은 보통 다음과 같은 정형화된 패턴으로 구성된다.</p>
<p><img alt="" src="https://velog.velcdn.com/images/kik328288/post/0854f21c-aacf-457c-bc9b-0c5706f44ab3/image.png" /></p>
<ul>
<li><strong>컨트롤러 (Controller):</strong> 웹 MVC의 컨트롤러 역할을 하며, 외부 요청을 받아 서비스나 리포지토리를 호출한다.</li>
<li><strong>서비스 (Service):</strong> 핵심 비즈니스 로직(예: 중복 가입 방지, 회원 등급 관리 등)을 구현한다.</li>
<li><strong>리포지토리 (Repository):</strong> 데이터베이스에 접근하여 도메인 객체를 저장하고 관리하는 역할을 한다.</li>
<li><strong>도메인 (Domain):</strong> 회원, 주문, 쿠폰 등 데이터베이스에 저장되고 관리되는 비즈니스 객체이다.</li>
</ul>
<h3 id="42-요구사항-분석">4.2 요구사항 분석</h3>
<hr />
<ul>
<li><strong>데이터:</strong> 회원 ID(시스템 식별자), 이름.</li>
<li><strong>기능:</strong> 회원 등록, 회원 조회.</li>
<li><strong>가정:</strong> 아직 데이터 저장소(DB)가 선정되지 않은 가상의 시나리오이다(RDB, NoSQL 등 고민 중).</li>
</ul>
<h3 id="43-설계-전략">4.3 설계 전략</h3>
<hr />
<ul>
<li>데이터 저장소가 확정되지 않았으므로, 나중에 구현체를 쉽게 교체할 수 있도록 <strong>인터페이스(Interface)</strong>를 정의하여 개발한다.</li>
<li>초기 개발 단계에서는 가벼운 <strong>메모리 기반의 저장소(<code>MemoryMemberRepository</code>)</strong>를 구현체로 사용한다.</li>
</ul>
