# Section 118 데이터 입/출력
## 데이터 입/출력의 개요
- 키보드로부터 데이터를 입력받아 화면이나 파일로 출력하는 것
## C언어
### 예제
- 다음은 키보드로 6과 4를 입력받아 두 수의 덧셈 결과를 출력하는 프로그램이다. 출력 결과를 확인하시오.
```C
#include <stdio.h>

main()
{
	int i, j, k;
	scanf("%d %d", &i, &j);
	k = i + j;
	printf("%d\n", k);
}
```
#### 예제 해설
```C
#inlcude <stdio.h>
```
- C언어에는 함수들을 모아 놓은 헤더 파일을 제공
- 헤더 파일을 사용하려면 include문을 이용해 선언한 후 사용해야 함
- “stdio.h” : 표준 입/출력과 관련된 함수를 정의해 놓은 헤더 파일
	- scanf() 함수와 printf() 함 수 등 포함
- 사용하는 함수에 따라 포함시켜야 할 헤더 파일은 달라짐

```C
main()
{
}
```
- 모든 C 프로그램은 반드시 main() 함수를 포함해야 실행 가능
- 프로그램을 중간에서 끝내는 특별한 제어문이 존재하지 않을 경우 무조건 “{“에서 시작해서 “}”에서 프로그램이 종료되어야 함

```C
int i, j, k;
```
- int : 정수 자료형을 의미하는 예약어
- i, j, k를 정수형 변수로 사용한다는 의미
- 자료형 : int, long, short, float 등
- 세미콜론(;)으로 문장이 종료되어야 함
	- C 프로그램에서는 줄 수와 상관없이 세미콜론이 나올 때 까지를 한 문장으로 인식하여 컴파일하기 때문에 문장 끝에 세미콜론이 없을 경우 컴파일 시 에러 발생

```C
scanf("%d %d", &i, &j);
```
- scanf() : 키보드로 입력 받는 대부분의 작업을 수행
- %d : 입력 받는 문자를 10진수로 받아들이겠다는 의미
- &(주소 연산자) : 변수에 사용한 이유는 scanf() 함수는 데이터를 입력 받을 때 주소를 사용하기 때문

```C
k = i + j;
```
- k에 i와 j를 더한 값을 저장하는 처리문
- 처리문은 입/출력을 제외하고 무엇인가를 처리하는 모든 문장 의미
	- 입/출력문, 제어문도 일종의 처리문이지만 별도로 구분하여 그렇게 부르기 때문에 이를 제외한 모든 문장을 처리문이라고 생각하기

```C
print("%d\n", k);
```
- printf() : C 프로그램에서 화면에 출력할 때 사용되는 함수
	- fprintf() : 파일에 출력할 때 사용되는 함수
- “\n” : 출력 내용을 제어하는 제어 문자
	- “\”와 그 뒤의 한 문자는 화면에 출력하지 않고 그 문자에 정해진 명령대로 출력물을 제어
## JAVA
### 예제
- 다음은 입력 받은 정수를 가지고 여러 연산을 수행한 후 출력하는 프로그램이다. 4가 입력되었다고 가정했을 때 출력 결과를 확인하시오.
```JAVA
import java.util.Scanner;

public class Test
{
	public static void main(String[] args)
	{
		Scanner scan = new Scanner(System.in);
		int a = scan.nextInt();
		System.out.printf("a * 3 = %d\n", a * 3);
		System.out.println("a / 2 = " + (a / 2));
		System.out.print("a - 1 = " + (a - 1));
		scan.close();
	}
}
```

```java
import java.util.Scanner;
```
- Java에는 메소드(함수)들을 모아놓은 클래스와 클래스를 모아놓은 패키지 제공
- import문을 사용해 패키지를 선언한 후 사용해야 함
- “java.util.Scanner”는 입력과 관련된 메소드(함수)를 정의해 놓은 Scanner 클래스가 java 패키지 안의 util 패키지에 있다는 것을 의미

```java
public class Test
```
- Java 프로그램은 아무리 작은 프로그램이라도 클래스 안에 속성(변수)과 메소드(함수)를 만들어서 실행
	- public class : 실행을 위한 클래스에 그대로 사용해야 하는 예약어
	- Test
		- 클래스 이름으로 사용자가 원하는 이름을 임의로 지정
		- 단, 첫 글자는 대문자를 지정하는 것이 관례

```java
public static void main(String[] args)
```
- 프로그램 실행 진입점 : 모든 Java 프로그램은 실행용으로 만든 클래스 안에 반드시 main() 메소드 포함해야 여기서부터 실행이 시작됨
	- public static : main() 메소드 앞에 반드시 써야하는 것
	- void : 반환값이 없다는 의미

```java
Scanner scan = new Scanner(System.in);
```
- Java에서 키보드로 입력 받은 값을 변수에 저장하려면 먼저 Scanner 클래스를 이용해 키보드로부터 값을 입력 받는 객체 변수를 생성한 후 사용
- Scanner 클래스의 객체 변수 scan을 키보드로 입력 받을 수 있도록 생성
	- Scanner : 입력에 사용할 객체 변수를 생성할 때 사용하는 클래스 이름으로 그대로 작성
	- scan : 객체 변수명으로 사용자가 임의로 작성
	- new : 객체 생성 예약어로 그대로 작성
	- Scanner() : 클래스 이름으로 ()를 붙여 그대로 작성
	- System.in
		- 표준 입력 장치, 즉 키보드를 의미
		- 키보드로부터 값을 입력 받는 객체 변수를 생성할 것이므로 그대로 작성

```java
int a = scan.nextInt();
```

	- scan.nextInt()

```java
System.out.printf("a * 3 = %d\n", a * 3);
```

```java
System.out.println("a / 2 = " + (a / 2));
```

```java
System.out.print("a - 1 = " + (a - 1));
```

# Section 119 제어문
### 제어문의 개요
- 서술된 순서에 따라 무조건 위에서 아래로 실행되는 프로그램의 순서를 변경할 때 사용하는 명령문
## 제어문 종류
### for문
```
for(초기값, 최종값, 증가값)
	실행할 문장;
```
- 초기값, 최종값, 증가값을 지정하는 수식을 이용해 정해진 횟수를 반복하는 제어문
### switch문
```
switch(수식)
{
	case 레이블1:
		실행할 문장1;
		break;
	case 레이블2:
		실행할 문장2;
		break;
	...
	
	default:
		실행할 문장3;
}
```
- 조건에 따라 분기할 곳이 여러 곳인 경우 간단하게 처리할 수 있는 제어문
### if문
```
if(조건식)
	조건이 참일 때 실행할 문장;
else
	조건이 거짓일 때 실행할 문장;
```
- 조건에 따라 실행할 문장을 다르게 하는 제어문
### while문
```
while(조건)
	실행할 문장;
```
- 조건이 참인 동안 실행한 문장을 반복 수행하는 제어문
### break와 continue
1. break : 반복문이나 switch문 안에서 break가 나오면 블록을 벗어남
2. continue : 반복문에서 continue가 나오면 continue 이후의 문장을 실행하지 않고 제어를 반복문의 처음으로 이동

## C언어
### 예제
- 다음 C언어로 구현된 프로그램의 실행 결과를 확인하시오.
```C
#inlcude <stdio.h>

main()
{
	int score[] = { 86, 53, 95, 76, 61 };
	char grade;
	char str[] = "Rank";
	
	for (int i = 0; i < 5; i++)
	{
		switch (score[i] / 10)
		{
			case 10:
			
			case 9:
				grade = "A";
				break;
			
			case 8:
				grade = "B";
				break;
			
			case 7:
				grade = "C";
				break;
			
			default:
				grade = "F";	
		}
		
		if (grade != "F")
			printf("%d is %c %s\n", i + 1, grade, str);
	}
}
```
#### 예제 해설
```c
int score[] = { 86, 53, 95, 76, 61 };
```
- 5개의 요소를 갖는 정수형 배열 score 선언
- 개수를 지정하지 않았으므로, 초기값으로 지정된 수만큼 배열의 요소 생성
	- int : 배열에 저장할 자료의 형
	- score : 사용할 배열의 이름으로 사용자가 임의로 지정
	- [] : 배열의 크기를 지정하는 것으로, 개수를 지정하지 않으면 초기값으로 지정된 수만큼 배열의 요소 생성

```c
char grade;
```
- 문자형 변수 grade 선언

```c
char str[] = "Rank";
```
- 문자형 배열 str을 선언하고 “Rank”로 초기화
- 문자열을 저장하는 경우 문자열의 끝을 의미하는 널 문자(“\0”)가 추가로 저장되며, 출력 시 널 문자는 표시되지 않음

```c
for (int i = 0; i < 5; i++)
```
- 반복 변수 i가 0에서 시작하여 1씩 증가하면서 5보다 작은 동안 for의 {} 부분을 반복 수행
- for의 {}을 5번 반복 수행
	- for : 반복문을 의미하는 예약어로 그대로 입력
	- i = 0 : 초기 값을 지정하는 수식
	- i < 5 : 최종값으로 지정하는 수식
	- i++ : 증가값으로 사용하는 수식

```c
switch (score[i] / 10)
```
- score[i]를 10으로 나눈 결과에 해당하는 숫자를 찾아가는 수식
- C, Java에서 정수형 변수에 저장된 값의 나눗셈은 결과도 정수가 됨
	- switch : switch문에서 사용되는 예약어로 그대로 입력
	- score[i] / 10 : case들의 값 중 하나를 도출하는 수식으로 변수 작성 가능
## Java
## 예제
- 다음 Java로 구현된 프로그램의 실행 결과를 확인하시오.
```java
public class Test 
{
	public static void main(String[] args)
	{
		String str = "agile";
		int x[] = { 1, 2, 3, 4, 5 };
		char y[] = new char[5];
		int i = 0;
		
		while (i < str.length())
		{
			y[i] = str.charAt(i);
			i++;
		}
		
		for (int p : x)
		{
			i--;
			System.out.print(y[i]);
			System.out.print(p + " ");
		}
	}
}
```
#### 예제 해설
```java
char y[] = new char[5];
```
- Java에서 배열은 객체로 취급되며, 객체 변수는 “new” 명령을 이용해서 생성
- 배열을 선언하면서 초기값을 지정하는 경우 선언하면서 지정하면 됨
	- char y[]
		- y는 문자형 배열 변수라는 의미
		- Java에서 대괄호를 자료형 바로 뒤에 적는 것을 선호하는데, C언어에서는 표기 불가능
	- new char[5] : 5개의 요소를 갖는 문자형 배열을 생성

```java
while (i < str.length())
```
- i가 str 변수의 길이보다 작은 동안 반복 수행
	- while : 반복문에서 사용되는 예약어 그대로 입력
	- (i < str.length()) : 조건으로, 참이나 거짓을 결과로 갖는 수식을 조건에 입력하며 참(1)을 직접 입력 가능
		- length() 메소드 : 변수의 크기를 반환하며 배열 클래스의 속성으로 배열 요소의 개수가 저장되어 있음

```java
y[i] = str.charAt(i);
```
- y[i]에 str의 i번째에 있는 문자를 저장
- charAt() 메소드 : 문자열에서 지정된 위치의 문자를 반환
# Section 120 포인터
## 포인터의 개요
- C언어에서는 변수의 주소를 포인터라 하고, 포인터를 저장할 수 있는 변수를 포인터 변수라 함
- 변수의 주소인 포인터는 출력할 수도 있고 포인터가 가리키는 곳에 값을 저장하거나 읽어 오는 등 다양한 조작 가능
## C언어
### 예제
- 다음 C언어로 구현된 프로그램의 실행 겨로가를 확인하시오.
```c
#include <stdio.h>

main()
{
	int a = 50;
	int *b = &a;
	*b = *b + 20;
	printf("%d, %d\n", a, *b);
	
	char *s;
	s = "gilbut";
	
	for (int i = 0; i < 6; i += 2)
	{
		printf("%c, ", s[i]);
		printf("%c, ", *(s + i));
		printf("%s\n", s + i);
	}
}
```
#### 예제 해설
```c
int a = 50;
```
- 정수형 변수 a를 선언하고 50으로 초기화
	- 주기억장치의 빈 공간 어딘가에 a라는 이름을 붙이고 그곳에 50을 저장

```c
int *b = &a;
```
- 정수형 변수가 저장된 곳의 주소를 기억할 포인터 변수 b를 선언하고, a의 주소로 초기화하며 b에는 a의 주소가 저장됨
	- int : 포인터 변수가 가리키는 곳에 저장되는 값의 자료형
	- \*b : 포인터 변수를 선언할 때는 변수명 앞에 \*을 사용
	- &a : 변수의 주소를 알아낼 때는 변수 앞에 번지 연산자 & 사용
- 변수 a의 주소가 b에 기억된다는 것은 b가 변수 a의 주소를 가리키고 있다는 의미

```c
*b = *b + 20;
```
- b가 가리키는 곳의 값에 20을 더함
- b가 가리키는 곳이 a이므로 a의 값도 변경됨
- 실행문에서 포인터 변수에 간접 연산자 \*를 붙이면 해당 포인터 변수가 가리키는 곳의 값을 의미
- b가 가리키는 곳의 값에 20을 더해 다시 b가 가리키는 곳에 저장하는데 그곳이 변수 a의 주소이므로 변수 a의 값도 저절로 변경되는 것

```c
char *s;
s = "gilbut";
```
- 문자형 변수가 저장된 곳의 주소를 기억할 포인터 변수 s 선언
- s는 주소를 기억하는 포인터 변수이므로 “gilbut”이 기억되는 것이 아니라 “gilbut”이라는 문자열이 메모리의 어딘가에 저장된 후 그 저장된 곳의 주소가 s에 기억됨

```c
# 가장 직관적이고 표준적인 배열 접근 방식입니다. 배열 `s`의 `i`번째 인덱스에 위치한 값을 가져옵니다.
printf("%c, ", s[i]);

# C언어 컴파일러 내부에서 배열 인덱싱을 처리하는 실제 방식입니다. `s[i]`와 `*(s + i)`는 C언어에서 완벽하게 동일한 문법입니다.

printf("%c, ", *(s + i));

# 단일 문자(%c)가 아닌 문자열(%s)을 출력하는 방식
# %s 형식 지정자는 특정 문자의 값을 요구하지 않고, 문자열이 시작되는 메모리 주소를 요구하며 그 주소부터 시작해서 널 문자를 만날 때까지의 모든 문자를 연속해서 출력
```

## 구조체의 개요
- 배열이 자료의 형과 크기가 동일한 변수의 모임이라면 구조체는 자료의 종류가 다른 변수의 모임
### 구조체의 정의
- 구조체를 정의한다는 것은 int나 char 같은 자료형을 하나 만드는 것을 의미
- 예시
```
struct sawon
{
	char name[10];
	char jikwi[10];
	int pay;
}
```
- struct : 구조체를 정의하는 예약어로 그대로 작성
- sawon
	- 구조체의 이름으로 사용자가 임의로 작성
	- 해당 이름의 자료형이 하나 생긴 것을 의미
- 멤버 : 일반 변수를 선언하는 것과 동일하게 필요한 필드들을 임의로 선언
### 구조체 변수의 선언
- 구조체 변수를 사용하려면 먼저 정의한 구조체에 대한 변수를 선언해야 함
- 예시
```
struct sawon ansan, *seoul;
```
- struct : 구조체 변수를 선언하는 예약어로 그대로 작성
- sawon ansan
	- 정의한 구조체 sawon 자료형으로 변수를 선언하는데 변수의 이름은 ansan이라는 의미
	- 사용자가 임의로 작성
- \*seoul : 구조체의 포인터 변수
### 구조체 멤버의 지정
- 구조체의 멤버는 모든 요소들이 개별적인 이름을 가지고 있으므로 구조체 멤버를 지정할 때는 구조체 변수와 함께 멤버 이름을 지정해야 함
- “.”에 의한 지정 : 구조체 일반 변수를 이용해 구조체 멤버를 지정하는 경우
```
ansan.name = "김한국";
ansan.jikwi = "대리";
ansan.pay = 4000000;
```
- “→”에 의한 지정 : 구조체 포인터 변수를 이용해 구조체 멤버를 지정하는 경우
```
seoul->name = "홍길동";
seoul->jikwi = "과장";
seoul->pay = 5000000;
```
- 구조체의 포인터 변수는 일반 포인터 변수처럼 \*사용하여 멤버 지정 가능
```
(*seoul).name = "홍길동";
(*seoul).jikwi = "과장";
(*seoul).pay = 500000;
```

## 추상 클래스와 형 변환의 개요
- 추상 메소드 : 자식 클래스에서 재정의해야만 사용할 수 있는 메소드
- 추상 클래스 : 이런 추상 메소드를 하나 이상 포함하는 클래스
- 클래스의 형 변환 : 이렇게 재정의한 메소드를 부모 클래스의 객체 변수를 통해 사용하려면 부모 클래스의 객체 변수를 선언할 때 자식 클래스의 생성자를 이용하는 것
	- 형 변환은 부모 클래스의 구조를 가지면서 자식 클래스의 속성이나 메소드로 재정의하고 싶을 때 사용하는 기법
	- 부모 클래스의 구조를 가져야 할 이유가 없다면 부모 클래스를 상속 받아 재정의하고 있는 자식 클래스의 구조로 객체 변수를 만들어 사용하면 됨
## 추상 메소드의 재정의
- 추상 메소드는 부모 클래스가 자식 클래스에게 주는 의무
- 부모 클래스의 상속 관계에 있다면 반드시 부모 클래스의 추상 메소드를 재정의해야 하며 그렇지 않을 경우 프로그램에 오류 발생
## Java 예제
- 다음 Java 프로그램의 실행 결과를 확인하시오.
```java

{
	String a = " is animal";
	abstract void look();
	void show()
	{
		System.out.println("Zoo");
	}
}

class Chicken extends Animal
{
	Chicken()
	{
		look();
	}
	
	void look()
	{
		System.out.println("Chicken" + a);
	}
	
	void display()
	{
		System.out.println("two wings");
	}
}

public class Test
{
	public static void main(String[] args)
	{
		Animal a = new Chicken();
		a.show();
	}
}
```

```java
abstract class Animal
{
}
```

	- abstract [클래스 정의부] : abstract는 추상 클래스를 정의하는 명령어로, 추상 클래스 정의 시 꼭 써야 하는 예약어
	- 추상 클래스는 내부에 실행 코드가 없는 추상 메소드를 포함하기 때문에 객체 변수의 생성자로 사용 불가능

```java
abstract void look();
```
- 추상 메소드 look() 정의
	- abstract [메소드 정의부] : abstract는 추상 메소드를 정의하는 명령어로 추상 메소드 정의 시 꼭 써야 하는 예약어
	- 추상 메소드는 선언만 있고 내부에 실행 코드가 없는 메소드로, 이후 상속 관계가 설정된 자식 클래스에서 재정의한 후 사용

```java
Animal a = new Chicken();
```
- Chicken 클래스의 생성자를 이용해 Animal 클래스의 객체 변수 a 선언
	- [부모 클래스명] [객체변수명] = new [자식클래스생성자()] : 부모 클래스의 객체 변수를 선언하면서 자식 클래스의 생성자를 사용하면 형 변환 발생

# Section 127 예외 처리
## 예외 처리 (Exception Handling)
- 예외가 발생했을 때 프로그래머가 해당 문제에 대비해 작성해 놓은 처리 루틴이 수행되도록 하는 것
	- 예외(Exception) : 프로그램의 정상적인 실행을 방해하는 조건이나 상태
- 예외가 발생했을 때 일반적인 처리 루틴은 프로그램을 종료시키거나 로그(Log)를 남기는 것
- C++, Ada, Java, 자바스크립트와 같은 언어에는 예외 처리 기능이 내장되어 있음
- 필요한 경우 조건문을 이용해 예외 처리 루틴을 작성
### 대표적인 예외의 원인
1. 컴퓨터 하드웨어 문제
2. 운영체제의 설정 실수
3. 라이브러리 손상
4. 사용자의 입력 실수
5. 받아들일 수 없는 연산
6. 할당하지 못하는 기억 장치 접근
## Java의 예외 처리
- 예외를 객체로 취급하며, 예외와 관련된 클래스를 java.lang 패키지에서 제공
- try ~ catch문을 이용해 예외 처리 수행
- try 블록 코드를 수행하다 예외가 발생하면 예외를 처리하는 catch 블록으로 이동하여 예외 처리 코드를 수행하므로 예외가 발생한 이후의 코드는 실행되지 않음
- catch 블록에서 선언한 변수는 해당 catch 블록에서만 유효
- try ~ catch문 안에 또 다른 try ~ catch문 포함 가능
- try ~ catch문 안에서는 실행 코드가 한 줄이라도 중괄호({ }) 생략 불가능
### 기본 형식
```java
try {
	예외가 발생할 가능성이 있는 코드;
} catch (예외객체1 매개변수) {
	예외객체1에 해당하는 예외 발생 시 처리 코드;
} catch (예외객체2 매개변수) {
	예외객체2에 해당하는 예외 발생 시 처리 코드;
} catch (예외객체n 매개변수) {
	예외객체n에 해당하는 예외 발생 시 처리 코드;
} catch (Exception 매개변수) {
	예외객체1~n에 해당하지 않는 예외 발생 시 처리 코드;
} finally {
	예외의 발생 여부와 관계없이 무조건 처리되는 코드;
}
```
## JAVA의 주요 예외 객체
1. ClassNotFoundException : 클래스를 찾지 못한 경우
2. NoSuchMethodException : 메소드를 찾지 못한 경우
3. FileNotFoundException : 파일을 찾지 못한 경우
4. InterruptIOException : 입/출력 처리가 중단된 경우
5. ArithmeticException : 0으로 나누는 등의 산술 연산에 대한 예외가 발생한 경우
6. IllegalArgumentException : 잘못된 인자를 전달한 경우
7. NumberFormatException : 숫자 형식으로 변환할 수 없는 문자열을 숫자 형식으로 변환한 경우
8. ArrayIndexOutOfBoundsException : 배열의 범위를 벗어난 접근을 시도한 경우
9. NegativeArraySizeException : 0보다 작은 값으로 배열의 크기를 지정한 경우
10. NullPointerException : 존재하지 않는 객체를 참조한 경우
