# JAVA
## Garbage Collector 모듈
- JAVA에서 힙(Heap)에 남아 있으나 변수가 가지고 있던 참조값을 잃거나 변수 자체가 없어짐으로써 더이상 사용되지 않는 객체를 제거해주는 역할을 하는 모듈
## JAVA의 예외(Exception)
- 실행 중에 발생할 수 있는 여러 상황들을 대비한 것
	1. 오동작이나 결과에 악영향을 미칠 수 있는 실행 시간 동안에 발생한 오류
	2. 배열의 인덱스가 그 범위를 넘어서는 경우 발생하는 오류
	3. 존재하지 않는 파일을 읽으려고 하는 경우에 발생하는 오류
- 문법 오류로 인해 발생한 것은 코드가 실행조차 되지 않으므로 예외로 처리 불가능

# 프로세스
- 프로세서가 할당되는 실체
- 디스패치가 가능한 단위
- 비동기적 행위를 일으키는 주체
- PCB를 가지며 PCB에는 프로세스의 현재 상태, 고유 식별자를 가지고 있음
## 프로세스 상태 종류
1. 제출 (Submit)
2. 접수 (Hold)
3. 준비 (Ready)
4. 실행 (Run/Running)
5. 대기 (Wait)
6. 종료 (Terminated, Exit)
# 프로세스 스케줄링
- CPU를 사용하려고 하는 프로세스들 사이의 우선순위를 관리하는 작업
- 처리율과 CPU 이용률을 증가시키고 오버헤드, 응답 시간, 반환 시간, 대기 시간을 최소화하기 위한 기법
## 프로세스 스케줄링 유형
### 선점형 스케줄링 (Preemptive Scheduling)
- 하나의 프로세스가 CPU를 차지하고 있을 때, 우선 순위가 높은 다른 프로세스가 현재 프로세스를 중단시키고 CPU를 점유하는 스케줄링 방식

| 종류        | 설명                                                                                                                                                                                                     |
| --------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| SRT       | Shortest Remaining Time First<br>1. 가장 짧은 시간이 소요되는 프로세스를 먼저 수행<br>2. 남은 처리 시간이 더 짧다고 판단되는 프로세스가 준비 큐에 생김<br>3. 언제라도 프로세스가 선점됨                                                                          |
| 다단계 큐     | MLQ : Multi Level Queue<br>작업들을 여러 종류의 그룹으로 분할<br>여러 개의 큐를 이용해 상위 단계 작업에 의한 하위 단계 작업이 선점 당하는 기법<br>각 큐는 독자적으로 스케줄링                                                                                     |
| 다단계 피드백 큐 | MFQ : Multi Level Feedback Queue<br>입출력 위주와 CPU 위주인 프로세스의 특성에 따라 큐마다 서로 다른 CPU 시간 할당량을 부여하는 기법<br>FCFS(FIFO)와 라운드 로빈 스케줄링 방식을 혼합한 기법<br>상위 단계에서 완료되지 못한 작업은 하위 단계로 전달되어 마지막 단계에서는 라운드 로빈 방식을 사용하는 기법   |
| 라운드 로빈    | RR : Round Robin<br>프로세스는 같은 크기의 CPU 시간을 할당<br>프로세스가 할당된 시간 내에 처리 완료 못할 경우 준비 큐 리스트의 가장 뒤로 보내지고, CPU는 대기 중인 다음 프로세스로 넘어가는 기법<br>시간 할당량이 너무 커지면 FCFS와 비슷하게 됨<br>시간 할당량이 너무 적으면 오버헤드가 커짐<br>시분할 시스템에서 사용 |
### 비선점형 스케줄링 (Non Preemptive Scheduling)

|                  종류                  | 설명                                                                                                       |
| :----------------------------------: | -------------------------------------------------------------------------------------------------------- |
|     FIFO<br>(First in First out)     | 먼저 도착한 프로세스가 먼저 처리됨<br>FCFS : First Come First Service                                                   |
|     SJF<br>(Shortest Job First)      | 실행 기간이 짧은 프로세스가 우선 처리<br>가장 낮은 대기 시간                                                                     |
| HRN<br>(Highest Response-ratio Next) | 실행 기간 긴 프로세스에 불리한 SJF 기법 보완 (기아 현상)<br>실행 기간이 짧거나 대기 시간이 긴 프로세스에 우선 부여<br>우선순위를 계산해 그 수치가 높은 순으로 우선순위 부여 |
|          우선순위<br>(Priority)          | 프로세스 별로 우선순위 부여<br>우선순위에 따라 CPU를 할당하는 기법<br>동일 순위는 FCFS 방식 사용                                            |
|          기한부<br>(Deadline)           | 작업들이 명시된 시간/기한 내에 완료되도록 계획하는 기법                                                                          |

# 디스크 스케줄링
- 사용할 데이터가 디스크 상의 여러 곳에 저장되어 있을 경우, 데이터를 액세스하기 위해 디스크 헤드를 움직이는 경로를 결정하는 기법
- 운영체제가 담당하고 디스크 스케줄링의 목적은 처리량 최대화, 응답 시간 최소화
## 디스크 스케줄링 종류

| 종류          | 설명                                                                                                                                                                                                                                                                        |
| ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| FCFS        | First Come First Served; FIFO: First in First Out<br>디스크 큐에 가장 먼저 들어온 트랙에 대한 요청을 가장 먼저 처리                                                                                                                                                                                 |
| SSTF        | Shortest Seek Time First<br>현재 위치에서 탐색 거리가 가장 짧은 트랙에 대한 요청을 먼저 서비스<br>일괄 처리 시스템에 유용                                                                                                                                                                                       |
| SCAN        | 1. 현재 헤드의 위치에서 진행 방향이 결정되면 탐색 거리가 짧은 순서에 따라 해당                                                                                                                                                                                                                            |
| C-SAN       | Circular SCAN<br>항상 바깥쪽에서 안쪽으로 움직이며 가장 짧은 탐                                                                                                                                                                                                                               |
| LOOK        | 엘리베이터 알고리즘<br>SCAN을 기초로 사용하는 기법<br>진행 방향으로 더 이상의 요청                                                                                                                                                                                                                       |
| N-STEP SCAN | SCAN 기법을 기초                                                                                  Shortest Latency Time First<br>섹터 큐잉 (Sector Queuing)<br>회전 지연 시간 최적화를 위해 구현된 기법<br>1. 디스크 헤드가 특정 실린더에 도착하면 그 실린더 내의 여러 트랙에 대한 요청들을 검사<br>2. 회전 지연 시간이 가장 짧은 요청부터 서비스 r>2 회전 |

# 운영체제
- OS (Operating System)
- 사용자가 컴퓨터의 하드웨어를 쉽게 사용할 수 있도록 인터페이스를 제공해주는 소프트웨어
- 하드웨어 구분
	- 중앙처리장치 : 컴퓨터의 장치를 제어하고 데이터를 처리
	- 기억 장치 : 데이터를 저장
	- 통신 장치 : 외부와의 통신을 담당
	- 입출력 장치 : 데이터 입력과 출력을 담당

## 운영체제 기능
### 제어 프로그램 (Control Program)

|                    종류                    | 기능                                                                      |
| :--------------------------------------: | ----------------------------------------------------------------------- |
|     감시 프로그램<br>(Supervisor Program)      | 각종 프로그램의 실행과 시스템 전체의 작동 상태 감시 및 감독                                      |
|   작업 제어 프로그램<br>(Job Control Program)    | 작업의 연속 처리를 위한 스케줄 및 시스템 자원 할당 담당<br>EX) Job Scheduler, Master Scheduler |
| 데이터 관리 프로그램<br>(Data Management Program) | 주기억 장치와 보조 기억 장치 사이의 데이터 전송과 보조 기억 장치의 자료 갱신 및 유지 보수 기능을 수행             |

|                     종류                     | 기능                                                                             |
| :----------------------------------------: | ------------------------------------------------------------------------------ |

|       서비스 프로그램<br>(Service Program)        | 효율성을 위해 사용 빈도가 높은 프로그램<br>EX) 링커(Linker), 정렬 프로그램, 라이브러리(Library)<br>유틸리티 프로그램 |
|        문제 프로그램<br>(Problem Program)        | 특정 업무 해결을 위해 사용자가 작성한 프로그램                                                     |
## 운영체제에서 커널의 기능

### 쉘(Shell)
- 사용자가 입력한 명령어 라인을 읽어 필요한 시스템 기능을 실행시키는 명령어 해석기
- 시스템과 사용자 간의 인터페이스 제공하며 여러 가지 내장 명령어 보유
- 사용자 명령을 해석하고 커널로 전달하는 기능 제공

#### export
- 기존 환경 변수의 값을 변경하거나, 새로운 환경 변수를 설정할 때 사용하는 명령어

- 변수를 export 시키면 전역 변수(Global)처럼 되어 끝까지 기억됨

# 모듈화 (Modularity)
- 시스템을 모듈로 분할하면 각각의 모듈을 별개로 만들고 수정할 수 있기 때문에 좋은 구조 생성 가능
- 모듈 간의 결합도가 약해야 독립적인 모듈이 될 수 있음
- 모듈 내 구성 요소들 간의 응집도가 강해야 좋은 모듈 설계
## 응집도 (Cohesion)
- 한 모듈 내에 있는 처리 요소들 사이의 기능적인 연관 정도를 표현
![](https://velog.velcdn.com/images/kik328288/post/0c280d42-3533-4966-ab60-fd53f9181c0d/image.png)

## 결합도 (Coupling)
- 모듈 간의 결합도를 약하게 하면 모듈 독립성이 향상된다.
- 결합도가 낮을수록 시스템 구현 및 유지 보수 작업이 쉬움
- 결합도를 최소화하고 응집도를 최대화해야 모듈의 재사용성이 올라감
- 모듈과 모듈 사이의 상호 의존 또는 연관 정도

### malloc 함수
- 실행 시간에 힙 메모리를 할당 받음
- malloc() 함수를 실행하여 메모리를 할당 받지 못하면 NULL 값 반환
- malloc() 함수로 할당 받은 메모리는 free() 함수를 통해 해제 가능
- 프로그램이 실행 중일 때 사용자가 직접 힙 영역에 메모리 할당 가능
### break 명령문
- switch~case 구문에서는 break 명령문을 생략해도 동일한 결과 획득 불가능
- continue 명령문과 함께 조건 분기 명령문에 해당하지 않음
- C언어에서 반복 처리를 위한 명령문이 아니라 가장 가까운 반복문이나 switch~case 구문을 탈출하는 역할

- 단항 연산자
- 메모리 영역의 시작 주소 반환

### 통신망
#### 근거리 통신망(LAN : Local Area Network)
- 집, 사물실, 학교 등 지리적으로 제한된 공간 내에서 컴퓨터, 서버, 프린터, 기타 네트워크 장치를 연결하여 데이터를 교환할 수 있도록 구성된 네트워크
- IEEE 802 LAN 표준안

| 표준안   | 설명             | 표준안    | 설명               |
| ----- | -------------- | ------ | ---------------- |
| 802.2 | 논리 링크 제어 (LLC) | 802.6  | 도시형 통신망 (MAN)    |
| 802.3 | CSMA/CD        | 802.8  | Fiber Optic LANS |
| 802.4 | Token Bus      | 802.9  | 종합 음성/데이터 네트워크   |
| 802.5 | Token Ring     | 802.11 | 무선 LAN           |

#### 무선랜 통신망(WLAN : Wireless Local Area Network)
- 유선 LAN과 무선 단말 사이를 무선 주파수를 이용하여 전송하는 네트워크

| 표준      | 설명                                                       |
| ------- | -------------------------------------------------------- |
| 802.11a | 5GHz 대역에서 54Mbps 속도를 제공                                  |
| 802.11b | 2.4GHz 대역에서 11Mbps 속도를 제공                                |
| 802.11e | QoS 강화를 위해 MAC 지원 기능을 채택<br>초고속 서비스 (IP 전화, 비디오)에 QoS 제공 |
| 802.11f | AP 간의 로밍 기능을 향상 시킨 표준                                    |
| 802.11g | 802.11b와 비슷하지만 속도가 향상<br>22Mbps 이상                       |
| 802.11i | 무선랜의 보안 기능 향상                                            |
#### 원거리 통신망(WAN : Wide Area Network)
- 지리적으로 넓은 지역에 걸쳐 여러 네트워크나 컴퓨터 시스템을 연결하는 네트워크
- LAN을 상호 연결 시키거나 국가망 또는 각 국가의 공중 통신망을 상호 접속시키는 국제 정보 통신망으로 설계 및 구축, 운용되는 네트워크
- 공중 통신망 사업자가 구축하고, 일반 대중 가입자들에게 보편적인 정보 통신 서비스 제공
- 거리에 제약이 없지만 다양한 경로를 지나 정보가 전달되기 때문에 LAN보다 속도가 느리고 에러율도 높음

|               기술                | 설명                                                                                                                                   |
| :-----------------------------: | ------------------------------------------------------------------------------------------------------------------------------------ |
|  전용 회선 방식<br>(Dedicated Line)   | 통신 사업자가 사전에 계약을 체결한 송신자/수신자끼리만 데이터를 교환하는 방식<br>EX) 점대점 프로토콜(PPP), HDLC, SDLC, HNAS 프로토콜                                              |
| 회선 교환 방식<br>(Circuit Switching) | 물리적 전용선을 활용해 데이터 전달 경로가 정해 진 후 동일 경로로만 전달되는 방식<br>데이터를 동시에 전송할 수 있는 양을 의미하는 대역폭이 고정되고 안정적인 전송률 확보 가능<br>EX) 점재점 프로토콜(PPP), ISDN 프로토콜 |
| 패킷 교환 방식<br>(Packet Switching)  | 전체 메시지를 각 노드가 수용할 수 있는 크기로 잘라서 보내는 방식<br>EX) X.25, 프레임 릴레이 프로토콜                                                                      |

| ------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |

| CSMA/CA | Carrier Sense Multiple Access with Collision Avoidance<br>반송파 감지 다중 접속 / 충돌 회피<br>IEEE 802.11 무선 LAN의 반이중 방식에서 사용하는 방식<br>데이터 전송 시, 매체가 비어 있음을 확인한 뒤 충돌을 피하기 위해서 임의 시간을 기다린 후 데이터를 전송하는 방법 |

## OSI 7계층별 데이터 전송 단위(PDU)
1. 물리 계층 : Bit
2. 데이터링크 계층 : Frame
3. 네트워크 계층 : Packet
4. 전송 계층 : Segment(TCP) / Datagram(UDP)
5. 세션 계층 : Data
6. 표현 계층 : Data
7. 응용 계층 : Data / Message
## 1계층 - 물리 계층
- 실제 장치들을 연결하는데 필요한 전기적, 물리적 세부 사항들을 정의하는 계층
- 통신 채널을 통해 전송되는 사용자 장치의 디지털 데이터를 통신 채널에 맞는 신호들로 변환
### 물리 계층 프로토콜

| 프로토콜   | 설명                                  |
| ------ | ----------------------------------- |
| RS-232 | PC와 음향 커플러, 모뎀 등을 접속하는 직렬 방식의 인터페이스 |
### 물리 계층 장비

| 장비                | 설명                                                                  |
| ----------------- | ------------------------------------------------------------------- |
| 허브<br>(Hub)       | 여러 대의 컴퓨터를 연결하여 네트워크로 보내거나, 하나의 네트워크로 수신된 정보를 여러 대의 컴퓨터로 송신하기 위한 장비 |
| 리피터<br>(Repeater) | 디지털 신호를 증폭시켜 주는 역할을 하여 신호가 약해지지 않고 컴퓨터로 수신되도록 하는 장비                 |
## 2계층 - 데이터 링크 계층
- 링크의 설정과 유지 및 종료 담당
- 물리적 연결을 이용해 신뢰성 있는 정보를 전송하려고 동기화, 오류 제어, 흐름 제어, 회선 제어 기능을 수행
- 네트워크 계층에 데이터를 전달하고, 물리 계층에서 발생할 수 있는 오류를 탐지하고 수정하는 기능 제공
- 링크의 설정과 유지 및 종료를 담당하며, 노드 간의 오류 제어와 흐름 제어 기능을 수행하는 계층
### 데이터 링크 계층 프로토콜

| 프로토콜    | 설명                                                                                                         |
| ------- | ---------------------------------------------------------------------------------------------------------- |
| HDLC    | High-level Data Link Control<br>점대점 방식이나 다중 방식의 통신에 사용되는 ISO에서 표준화<br>동기식 비트 중심의 데이터 링크 프로토콜               |
| PPP     | Point-to-Point Protocol<br>네트워크 분야에서 두 통신 노드 간의 직접적인 연결을 위해 일반적으로 사용되는 데이터 링크 프로토콜                         |
| 프레임 릴레이 | Frame Relay<br>프로토콜 처리를 간략화하여 단순히 데이터프레임들의 중계 기능과 다중화 기능만 수행함으로써 데이터 처리 속도의 향상 및 전송 지연을 감소시킨 고속의 데이터 전송 기술 |
| ATM     | Asynchronous Transport Mode<br>정보 전달의 기본 단위를 53바이트 셀 단위로 전달<br>비동기식 시분할 다중화 방식의 패킷형 전송 기술                  |
### 데이터 링크 계층 장비

| 장비              | 설명                                                                                                                               |
| --------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| 스위치<br>(Switch) | 느린 전송 속도의 브리지, 서브의 단점을 개선<br>출발지에서 들어온 프레임을 목적지 MAC 주소 기반으로 빠르게 전송<br>EX) Cut-Through 방식, Store-and-Forward 방식, Fragment Free 방식 |
| 브리지<br>(Bridge) | 두 개의 근거리 통신망을 서로 연결해주는 통신망 연결 장치                                                                                                 |
#### 스위치 장비 주요 기술 요소

| 요소   | 설명                                                                                                                          |
| ---- | --------------------------------------------------------------------------------------------------------------------------- |
| VLAN | Virtual Local Area Network<br>물리적 배치와 상관없이 논리적으로 LAN을 구성하여 Broad-cast Domain을 구분할 수 있게 해주는 기술<br>접속된 장비들의 성능 향상 및 보안성 증대 효과 |
| STP  | Spanning Tree Protocol<br>2개 이상의 스위치가 여러 경로로 연결될 때, 무한 루프 현상을 막기 위해서 우선순위에 따라 1개의 경로로만 통신하도록 하는 프로토콜                        |
### 데이터 링크 계층의 오류 제어
- 오류 제어 : 데이터 전송 시 감쇠, 왜곡, 잡음에 의해 생성된 오류를 검출하고 정정하는 기능으로 데이터 전송의 신뢰성을 위해 필요한 기능

| 종류            | 설명                                                                                                                                                              |
| ------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 전진(순방향) 오류 수정 | FEC : Forward Error Correction<br>데이터 전송 과정에서 발생한 오류를 검출하여 검출된 오류를 재전송 요구 없이 스스로 수정하는 방식<br>EX) 해밍 코드 방식, 상승 코드 방식                                              |
| 후진(역방향) 오류 수정 | BEC : Backward Error Correction<br>데이터 전송 과정에서 오류가 발생하면 송신 측에 재전송을 요구하는 방식<br>오류 검출 방식 : 패리티 검사, CRC, 블록합 검사<br>오류 제어 : 자동 반복 요청(ARQ: Automatic Repeat reQuest) |
- 해밍(Hamming) 코드 방식
	- 수신 측에서 오류가 발생한 비트를 찾아 재전송을 요구하지 않고 자신이 직접 오류를 수정하는 방식
	- 1비트의 오류 수정 가능
- 상승 코드(부호) 방식
	- 여러 개의 비트의 오류가 있더라도 한계값(경계값), 순차적 디코딩을 이용해 모두 수정할 수 있는 방식
- 패리티 검사 (Parity Check)
	- 7~8개의 비트로 구성되는 전송 문자에 패리티 비트를 추가하여 오류를 검출하는 방식
- CRC (순환 잉여 검사 : Cycle Redundancy Check)
	- 다항식을 통해 산출된 CRC 값을 토대로 오류를 검사하는 방식
	- 집단 오류를 해결하기 위한 방식
- 블록합 검사 (Block Sum Check)
	- 프레임의 모든 문자로부터 계산되는 잉여 패리티 비트들을 사용하는 이차원(가로/세로) 패리티 검사 방식
#### 자동 반복 요청 방식(ARQ) 종류

| 종류                      | 설명                                                                                                                                                                                                                                                         |
| ----------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Stop-and-Wait ARQ 방식    | 한 개의 프레임을 전송하고, 수신 측으로부터 ACK 및 NAK 신호를 수신할 때까지 정보 전송을 중지하고 기다리는 방식<br>송신 측이 수신 측으로부터 ACK를 받으면 다음 프레임을 전송하고 NAK를 받으면 재전송<br>데이터프레임의 정확한 수신 여부를 매번 확인하면서 다음 프레임을 전송해 나가는 가장 간단한 오류 제어 방식<br>구현이 간단하고 송신 측에서 최대 프레임 크기의 버퍼 1개만 있어도 됨<br>전송 시간이 긴 경우 전송 효율이 저하 |
| Go-Back-N ARQ 방식        | 데이터프레임을 연속적으로 전송하는 과정에서 NAK를 수신할 경우, 오류가 발생한 프레임 이후에 전송된 모든 데이터프레임을 재전송하는 방식                                                                                                                                                                               |
| Selective Repeat ARQ 방식 | 연속적으로 데이터프레임을 전송하고 에러가 발생한 데이터프레임만 재전송하는 방식                                                                                                                                                                                                                |
## 3계층 - 네트워크 계층
- 다양한 길이의 패킷을 네트워크들을 통해 전달하고, 그 과정에서 전송 계층이 요구하는 서비스 품질(QoS)을 위한 수단을 제공하는 계층
- 라우팅, 패킷 포워딩, 인터 네트워킹 수행
	- 라우터 또는 교환기의 경우 패킷 전달을 위해 경로를 지정하거나 교환 기능 제공
- 패킷에 발신지와 목적지의 논리 주소를 추가하고, 패킷을 발신지로부터 최종 목적지까지 전달하는 책임을 짐
	- 패킷에 발신지와 목적지의 논리 주소 추가
### 네트워크 계층 프로토콜

| 프로토콜     | 설명                                                                                                                                                                                                                             |
| -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| IP       | Internet Protocol<br>컴퓨터 네트워크에서 데이터를 전달하기 위한 규칙과 방법을 정의<br>인터넷의 기본적인 통신 프로토콜<br>데이터를 패킷으로 나누어 네트워크를 통해 목적지로 전달하는 기능                                                                                                            |
| ARP      | Address Resolution Protocol<br>IP 네트워크 상에서 논리 주소(IP 주소)를 물리 주소(MAC 주소)로 변환                                                                                                                                                     |
| RARP     | Reverse Address Resolution Protocol<br>물리 주소를 논리 주소로 변환하는 프로토콜                                                                                                                                                                 |
| ICMP     | Internet Control Message Protocol<br>IP의 동작 과정에서의 전송 오류가 발생하는 경우에 대비해 오류 정보를 전송하는 목적으로 사용하는 프로토콜<br>메시지 형식은 8바이트의 헤더와 가변 길이의 데이터 영역으로 분리<br>수신지 도달 불가 메시지는 수신지 또는 서비스에 도달할 수 없는 호스트를 통지하는 데 사용<br>ICMP 프로토콜을 사용해서 ping 유틸리티 구현 |
| IGMP     | Internet Group Management Protocol<br>호스트 컴퓨터와 인접 라우터가 멀티캐스트 그룹 멤버십을 구성하는 데 사용하는 통신 프로토콜<br>화상 회의, IPTV 등에서 활용<br>그룹 가입, 멤버십 감시/응답/탈퇴 기능 존재                                                                                    |
| 라우팅 프로토콜 | 네트워크에서 데이터 패킷이 목적지에 도달하는 최적의 경로를 찾기 위해 라우터 간에 정보를 교환하는 규칙과 방법을 정의한 프로토콜                                                                                                                                                        |
### 네트워크 계층 장비

| 장비     | 설명                                                                                                                                                                           |
| ------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 라우터    | 서로 다른 네트워크 대역에 있는 호스트들 상호 간에 통신할 수 있게 해주는 장비<br>패킷의 목적지를 추출하여 그 목적지에 대한 최적의 경로 설정<br>해당 경로를 따라 데이터 패킷을 다음 장치로 전송시키는 장비<br>라우팅 프로토콜은 경로 설정을 해 원하는 목적지까지 지정된 데이터가 안전하게 전달되도록 함 |
| L3 스위치 | 3계층에서 네트워크 단위들을 연결하는 통신 장비<br>IP 계층에서의 스위칭을 수행하여 외부로 전송<br>L2 기능 + 경로 제어 기능 + 고속 라우팅 기능 수행                                                                                   |
### 라우팅 프로토콜
- 데이터 전송을 위해 목적지까지 갈 수 있는 여러 경로 중 최적의 경로를 설정해주는 라우터 간의 상호 통신 규약

| 분류     | 설명                                                                                                                                                                                                                                             |
| ------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 정적 라우팅 | 네트워크 관리자가 수동으로 라우팅 경로를 설정하고 구성하는 방식<br>라우터에 목적지 네트워크로의 경로 직접 설정 필요                                                                                                                                                                             |
| 동적 라우팅 | 라우터들이 라우팅 프로토콜을 사용하여 네트워크의 변경 사항을 자동으로 감지하고 라우팅 경로를 동적으로 업데이트하는 방식<br>내부 라우팅 프로토콜 (IGP : Interior Gateway Protocol) : 동일한 AS 내의 라우팅에 사용되는 프로토콜<br>외부 라우팅 프로토콜 (EGP : Exterior Gateway Protocol) : 서로 다른 AS 간 라우팅 프로토콜로 게이트웨이 간의 라우팅에 사용되는 프로토콜 |

## 4계층 - 전송 계층
- 상위 계층들이 데이터 전달의 유효성이나 효율성을 생각하지 않도록 해주면서 종단 간의 사용자들에게 신뢰성 있는 데이터를 전달하는 계층
- 단말기 사이에 오류 수정과 흐름 제어를 수행하여 신뢰성 있고 명확한 데이터를 전송
- 순차 번호 기반의 오류 제어 방식 사용
- 종단 간 통신을 다루는 최하위 계층으로 종단 간 신뢰성 있고 효율적인 데이터 전송
### 전송 계층 프로토콜

| 프로토콜 | 설명                                                                                                                                            |
| ---- | --------------------------------------------------------------------------------------------------------------------------------------------- |
| TCP  | Transmission Control Protocol<br>전송 계층에 위치하면서 근거리 통신망이나 인트라넷, 인터넷에 연결된 컴퓨터에서 실행되는 프로그램 간에 일련의 옥텟을 안정적으로, 순서대로, 에러 없이 신뢰성 있게 교환할 수 있게 해주는 프로토콜 |
| UDP  | User Datagram Protocol<br>비연결성이며 신뢰성이 없으며, 순서화되지 않은 데이터그램 서비스를 제공하는 전송 계층(4계층)의 프로토콜                                                          |
#### TCP 특징
- 신뢰성 있는 연결 지향성 전달 서비스 제공 (연결형 프로토콜)
- 순서 제어, 에러 제어, 흐름 제어 기능 제공
- 전이중 서비스와 스트림 데이터 서비스 제공
	- 양방향 가상 회선 제공
- 전송 데이터와 응답 데이터 함께 전송 가능
- 메시지를 캡슐화와 역캡슐화 수행
- 서비스 처리를 위해 다중화, 역다중화 이용

| 특징                            | 설명                                                                                                          |
| ----------------------------- | ----------------------------------------------------------------------------------------------------------- |
| 신뢰성 보장                        | 패킷 손실, 중복, 순서 바뀜 등이 없도록 보장<br>TCP 하위 계층인 IP 계층의 신뢰성 없는 서비스를 보완하여 신뢰성 제공<br>신뢰성 있는 스트림 전송 기능 제공              |
| 연결 지향적 특징                     | 같은 전송 계층의 UDP가 비연결성인 것과 달리, TCP는 연결 지향적<br>양단 간 애플리케이션/프로세스는 TCP가 제공하는 연결성 회선 통해 통신<br>신뢰성 있는 연결 지향형 전달 서비스 |
| 흐름 제어<br>(Flow Control)       | 흐름 제어 기능을 활용해 송신 및 수신 속도를 일치시킴                                                                              |
| 혼잡 제어<br>(Congestion Control) | 네트워크가 혼잡하다고 판단될 때는 혼잡 제어 기법을 사용해 송신율을 감속                                                                    |
#### UDP 특징

| 특징                    | 설명                                                                                                                               |
| --------------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| 비연결성<br>비신뢰성          | 데이터그램 지향의 전송 계층용 프로토콜<br>논리적인 가상 회선 연결 필요 없음<br>메시지가 제대로 도착했는 지 확인하지 않음 = 확인 응답 없음<br>검사 합을 제외한 특별한 오류 검출 및 제어 없음<br>오류 제어 거의 없음 |
| 순서화되지 않은 데이터그램 서비스 제공 | 수신된 메시지의 순서를 맞추지 않음<br>흐름 제어를 위한 피드백 제공하지 않음                                                                                     |

#### TCP/UDP 헤더 구조
- 순서 번호(Sequence Number) : 전달하는 바이트마다 번호 부여
- 수신 번호 확인 (Acknwoledgement Number) : 상대편 호스트에서 받으려는 바이트의 번호 정의
- 체크섬 (Checksum) : IPv4 헤더 구조 중, 도착한 패킷에 대한 오류 여부를 체크하기 위해 존재하는 요소
- 위도우 크기는 송수신 측의 버퍼 크기로 최대 크기는 64bit

# IP (Internet Protocol)
- 송신 호스트와 수신 호스트가 패킷 교환 네트워크에서 정보를 주고받는 데 사용하는 정보 위주의 규약
- OSI 3계층인 네트워크 계층에서 호스트의 주소 지정과 패킷 분할 및 조립 기능 담당
- IP의 정보는 패킷 혹은 데이터그램이라고 하는 덩어리로 나누어서 전송

| 특징               | 설명                                                                                                                                                                                                              |
| ---------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 비신뢰성<br>비연결형     | Unrealible, Connection less<br>흐름에 관여하지 않기 때문에 보낸 정보가 제대로 갔는 지 보장하지 않음<br>패킷 전송과 정확한 순서를 보장하려면 TCP 프로토콜과 같은 IP의 상위 프로토콜을 이용해야 함<br>신뢰성이 중요한 전송에는 IP+TCP 활용                                                      |
| 에러 제어와 흐름 제어가 없음 | 정보의 전송이 가장 중요<br>보낸 정보의 내용을 보장해주지 않음                                                                                                                                                                            |
| 대표적인 IP 주소 체계    | 패킷은 에러 검출만 하고 만일 발견되면 폐기                                                                                                                                                                                        |
| IP 크기 제한         | 현재 인터넷에서 사용하는 표준 프로토콜은 IPv4<br>주소 공간 고말 문제를 겪고 있어서 IPv6가 점차 확산<br>IP 헤더의 크기 : 20~40바이트<br>IP 패킷은 네트워크가 수용할 수 있는 크기로 분할되어야 하는데, 이를 단편화(Fragmentation)이라 함<br>인터넷으로 데이터 전송 시에는 IP 패킷을 잘라서 보내고, 이런 단편화는 MTU 때문에 필요 |
## IP 프로토콜에서 사용하는 필드
1. Header Length
	- IP 프로토콜의 헤더 길이를 32비트 워드 단위로 표시
2. Packet Length
	- IP 헤더를 포함한 패킷 전체의 길이 표현
	- 최대 크기는 65,535 (2^16-1) 바이트
3. Time to Live
	- 송신 호스트가 패킷을 전송하기 전 네트워크에서 생존할 수 있는 시간을 지정
4. Version Number
	- IP 프로토콜의 버전 번호 표시
# IPv4 / IPv6 특징
![](https://velog.velcdn.com/images/kik328288/post/7eb4a57e-beab-4e72-8926-7b51bc927b15/image.png)

- IPv6
	- IPv6 패킷 헤더는 40바이트의 고정된 길이를 가지므로 IPv4처럼 Header Length Field가 별도로 필요하지 않음
	- 주소 자동 설정 기능을 통해 손쉽게 이용자의 단말을 네트워크에 접속 시키는 것 가능
- IPv4
	- 호스트 주소를 자동으로 설정하여 유니캐스트 지원
	- 클래스별로 네트워크와 호스트 주소의 길이가 다름
## IPv4 네트워크 클래스

| 클래스 | 설명                                                                                                                                                                                                                           |
| --- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| A   | 가장 높은 단위의 클래스<br>1~126 (0, 127로 시작된 IP는 예약됨) 범위의 IP 주소 보유<br>두 번째, 세 번쨰, 네 번째 단위의 세 숫자는 A Class가 자유롭게 네트워크 사용자에게 부여 가능한 IP<br>범위 : 0.0.0.0 ~ 127.255.255.255 (서브넷 마스크 : 255.0.0.0)                                           |
| B   | 두 번째로 높은 단위의 클래스<br>첫 번째 단위의 세 숫자는 128~191 가운데 하나 보유<br>두 번째 단위의 세 숫자는 B Class가 접속할 수 있는 네트워크 지시<br>범위 : 128.0.0.0 ~ 191.255.255.255 (서브넷 마스크 : 255.255.0.0)                                                                 |
| C   | 최하위 클래스<br>IP 구성에서 첫 번째 단위의 세 숫자는 192~223 가운데 하나 보유<br>두 번째와 세 번째 단위의 세 숫자는 C Class가 접속할 수 있는 네트워크 지시<br>C Class가 자유로이 부여할 수 있는 IP는 마지막 네 번째 단위의 254개 (2개는 예약)<br>범위 : 192.0.0.0 ~ 223.255.255.255 (서브넷 마스크 : 255.255.255.0) |
| D   | 멀티캐스트 용도로 예약된 주소<br>범위 : 224.0.0.0 ~ 239.255.255.255                                                                                                                                                                         |
| E   | 연구를 위해 예약된 주소<br>범위 : 240.0.0.0 ~ 255.255.255.255                                                                                                                                                                            |

# IPSec (IP Security)
- 통신 세션의 각 IP 패킷을 암호화하고 인증하는 안전한 인터넷 프로토콜 통신을 위한 인터넷 프로토콜
- 암호화 수행 시 양방향 암호화 지원
- ESP는 발신지 인증, 데이터 무결성, 기밀성 모두 보장
- 운영 모드는 Tunnel 모드와 Transport 모드로 분류
- AH는 발신지 호스트를 인증하고, IP 패킷의 무결성을 보장
- 전송 모드 (Transport) : 전송 계층과 네트워크 계층 사이에 전달되는 payload 보호
- 터널 모드 (Tunnel) : IPSec이 IP 헤더를 포함한 IP 계층의 모든 것을 보호

# 프로세스 (Process)
- 프로세서(CPU)에 의해 처리되는 사용자/시스템 프로그램
- 프로세스 제어 블록 (PCB, Process Control Block)에 프로세스 식별자, 상태 등의 중요 정보를 저장
- 각 프로세스 실행 시 고유 PCB 생성, 작업 완료 후 PCB 제거
![](https://velog.velcdn.com/images/kik328288/post/464a0fe3-bee8-4996-9929-e3fdfc6d5a2b/image.png)

- Dispatch : 준비 상태에서 대기 중인 프로세스가 실행 상태로 전이되는 과정
- Wake up : 입출력 작업이 완료되어 프로세스가 대기 상태에서 준비 상태로 전이
- Context Switching (문맥 교환) : 이전 프로세스의 상태 레지스터 내용을 보관하고 다음 프로세스의 레지스터를 적재하는 과정
## 프로세스 상태
- 운영체제에서 하나의 프로그램이 실행되는 동안 나올 수 있는 상태 변화 단계

| 상태                  | 설명                                                                                             |
| ------------------- | ---------------------------------------------------------------------------------------------- |
| 생성 상태<br>(Create)   | 사용자에 의해 프로세스가 생성된 상태                                                                           |
| 준비 상태<br>(Ready)    | CPU를 할당 받을 수 있는 상태<br>준비 리스트는 각각 우선순위를 부여하여 가장 높은 우선 순위를 갖는 프로세스가 다음 순서에 CPU를 할당 받음            |
| 실행 상태<br>(Running)  | 프로세스가 CPU를 할당 받아 동작 중인 상태                                                                      |
| 대기 상태<br>(Waiting)  | 프로세스 실행 중 입출력 처리 등으로 인해 CPU를 양도<br>→ 입출력 처리가 완료될 때까지 대기 리스트에서 기다리는 상태<br>대기 리스트는 우선순위가 존재하지 않음 |
| 완료 상태<br>(Complete) | 종료 상태 (Terminated; Exit)<br>프로세스가 CPU를 할당 받아 주어진 시간 내에 완전히 수행을 종료한 상태                          |
## 프로세스 상태 전이

# 스레드 (Thread)
- 프로세스 내 작업 단위 / 프로그램 단위
- 프로세스보다 가벼운, 독립적으로 수행되는 순차적인 제어 흐름/실행 단위
- 프로세스에서 실행 제어만 분리한 실행 단위
- 한 개의 프로세스에는 여러 개의 스레드 가능
## 스레드 종류

| 종류  | 설명                |
| --- | ----------------- |
| 단일  | 하나의 프로세스에 하나의 스레드 |
| 다중  | 하나의 프로세스에 복수의 스레드 |
## 스레드의 장점
1. 한 프로세스 내 복수의 스레드 생성하여 작업 병행성 향상
2. 응답 시간 단축
3. 기억 장소 낭비 최소화
4. 프로세스 간 통신 향상
5. 운영체제 성능 및 프로그램 처리 효율 향상
6. 접근 가능한 기억 장치 사용

| 수준     | 설명                                                                                                                                                                                                      |
| ------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 사용자 수준 | 사용자가 만든 라이브러리를 이용한 스레드 운용<br>스케줄링 결정이나 동기화 위해 커널 모드로 전환하지 않음<br>→ 인터럽트가 발생할 때 오버헤드가 적음<br>사용자 영역 스레드에서 행동<br>→ OS 스케줄러의 문맥 교환이 없음<br>속도 빠르지만 구현이 어려움                                                    |
| 커널 수준  | 스레드를 생성하고 스케줄링하는 주체가 커널<br>운영체제의 커널에 의한 스레드 운용<br>커널이 각 스레드를 개별적으로 관리 가능<br>다른 스레드가 입출력 작업이 끝날 때까지 다른 스레드 사용해 다른 작업 진행 가능<br>사용자 모드 → 커널 모드로의 전환이 많기 때문에 오버헤드 많이 발생<br>구현이 쉽지만 속도가 느림<br>안정성과 다양한 기능 제공 |
### 사용자 수준 스레드가 커널 스레드 비교
1. 커널 수준 스레드
	- 한 프로세스가 운영체제를 호출할 때 전체 프로세스가 대기할 필요 없기 때문에 시스템의 성능 향상 가능
	- 동시에 여러 스레드가 커널에 접근할 수 있으므로 여러 스레드가 시스템 호출을 동시에 사용 가능
	- 커널 모드로의 전환 없이 스레드 교환이 가능하므로 오버헤드 감소
2. 사용자 수준 스레드
	- 각 스레드를 개별적으로 관리할 수 있으므로 스레드의 독립적인 스케줄링 가능
# 교착 상태 (Dead Lock)
- 둘 이상의 프로세스들이 서로 점유하고 있는 자원을 요구하며 무한정 대기하고 있는 상태
## 교착 상태 필요 충분 조건

|   조건    | 설명                                                                                                                                                                                                     |
| :-----: | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
|  상호 배제  | 한 번에 한 개의 프로세스만이 공유 자원을 사용 가능<br>Mutual Exclusion                                                                                                                                                      |
| 점유 및 대기 | 최소한 하나의 자원을 점유하면서 다른 프로세스에 할당된 자원을 추가로 점유하기 위해 대기하는 프로세스 존재<br>이미 자원을 가진 프로세스가 다른 자원의 할당을 요구하는 동안 가진 자원을 계속 점유<br>프로세스가 수행되기 전에 필요한 모든 자원을 할당시켜준다.<br>자원이 점유되지 않은 상태에서만 자원을 요구하도록 함<br>(Hold and Wait) |
|   비선점   | 다른 프로세스에 할당된 자원은 끝날 때까지 강제로 선점 불가능<br>(Non-preemption)                                                                                                                                                 |
|  환형 대기  | 프로세스들이 원형으로 구성되어 있어 앞/뒤 프로세스 자원을 요구<br>이미 자원을 가진 프로세스가 앞이나 뒤의 프로세스의 자원을 요구<br>(Circular Wait)                                                                                                          |
## 교착 상태 해결 방법

| 방법                 | 설명                                                                                                                                          |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------- |
| 예방<br>(Prevention) | 교착 발생 4가지 조건 중 하나 이상을 사전에 제거하여 예방<br>교착 상태가 발생하지 않도록 사전에 시스템을 제어하는 방법<br>일반적으로 자원의 낭비가 가장 심한 기법                                             |
| 회피<br>(Avoidance)  | 발생된 교착 상태를 적절하게 피해가는 방법<br>교착 상태 발생 가능성을 인정하고 발생 시점에 가능성을 피해가는 방법<br>* 은행원 알고리즘 (Banker’s Algorithm) : 은행에서 모든 고객의 요구가 충족되도록 현금을 할당하는 데서 유래 |
| 발견<br>(Detection)  | 교착 상태에 있는 프로세스 및 자원을 발견 / 점검<br>교착 상태가 발생했는 지 검사해 교착 상태인 프로세스/자원을 발견하는 방법                                                                   |
| 회복<br>(Recovery)   | 교착 상태 유발 프로세스 종료<br>프로세스 내 할당된 자원 선점                                                                                                        |
# 페이징 기법 (Paging)
- 가상 기억 장치에 보관된 프로그램과 주기억장치의 영역을 동일 크기로 분할 후 나눠진 프로그램(페이지)을 동일하게 나눠진 주기억장치의 영역(페이지 프레임)에 배치하여 실행하는 기법
- 내부 단편화 발생 가능성 존재
## 페이징 기법 종류

| 크기        | 설명                                                                           |
| --------- | ---------------------------------------------------------------------------- |
| 작은 페이지 크기 | ==페이지 단편화 감소 / 페이지 이동 시간 감소 / 워킹 셋 유지<br>맵 테이블 크기 커지며 매핑 속도 감소 / 입출력 시간 증가== |
| 큰 페이지 크기  | 맵 테이블 크기 작아지며 매핑 속도 증가 / 입출력 시간 감소<br>페이지 단편화 증가 / 페이지 이동 시간 증가 / 불필요 데이터 적재 |
- 페이지가 작아질수록 발생하는 현상
	- 기억 장소 이용 효율 증가
	- 입출력 시간 증가
	- 내부 단편화 감소
	- 페이지 맵 테이블 크기 증가
## 페이지 교체 알고리즘
- 주기억장치에 있는 프로세스 중 어떤 프로세스를 제거할 것인지 결정하는 기법
- 새로운 페이지를 할당하기 위해 현재 할당된 페이지 중 어느 것과 교체할 지 결정하는 방법

|  기법  | 설명                                                                                                                                                                                                                                   |
| :--: | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| OPT  | 앞으로 갖아 오랫동안 사용하지 않을 페이지를 교체<br>페이지 부재 횟수가 가장 적게 발생하는 효율적인 교체 알고리즘                                                                                                                                                                    |
| FIFO | First-in First-out (선입선출)<br>가장 먼저 들어와서 가장 오래 있었던 페이지를 교체<br>1. 각 페이지가 주기억 장치에 적재될 때마다 그때 시간을 기억시킴<br>2. 가장 먼저 들어와 가장 오래 있던 페이지를 교체하는 기법                                                                                             |
| LRU  | Least Recently Used<br>최근에 가장 오랫동안 사용하지 않은(오래 전에 사용된) 페이지를 교체<br>프로그램의 지역성의 원리에 따라 최근에 참조된 페이지는 앞으로도 참조될 가능성 크며 최근에 참조되지 않은 페이지는 앞으로도 참조되지 않을 가능성이 크다는 전제                                                                            |
| LFU  | Least Frequently Used<br>사용 빈도가 가장 적은 페이지를 교체<br>활발한 페이지는 교체 X                                                                                                                                                                       |
| NUR  | Not Used Recently<br>LRU와 비슷한 알고리즘<br>최근에 사용하지 않은 페이지를 교체하는 기법<br>최근에 사용되지 않은 페이지는 앞으로도 사용되지 않을 가능성이 크다는 것 전제<br>→ LRU에서 나타나는 시간적인 오버헤드 감소 가능<br>최근의 사용 여부를 확인하기 위해 페이지마다 참조 비트와 변형 비트 사용<br>시간 오버헤드를 줄이기 위해 각 페이지마다 참조 비트와 변형 비트를 둠 |
| MFU  | Most Frequently Used<br>참조 횟수가 가장 많은 페이지 교체                                                                                                                                                                                          |

# 스레싱 (Thrashing)
- 프로세스 처리 시간보다 페이지 교체 소요 시간이 더 많아지는 상태
- 다중 프로그래밍 다발 시 스레싱 발생 → CPU 이용률 저하
- 오류율이 클수록 스레싱이 많이 발생하고, 스레싱으로 인해 전체 시스템의 성능 및 처리율 저하
- 페이지 부재가 계속 증가하면 기억 장치 접근 시간도 증가
![](https://velog.velcdn.com/images/kik328288/post/ea3a9cf9-b2a2-4da7-8bb4-70e9551b048b/image.png)

## 스레싱 방지 방법
1. 다중 프로그래밍 적정 수준 유지
2. 페이지 부재 빈도 조절
3. 워킹 셋 유지
4. 일부 프로세스 중단
5. CPU 성능에 대한 자료의 지속적인 관리 및 분석
## 워킹 셋 (Working Set)
- 일정 시간 동안 자주 참조하는 페이지들을 주기억장치에 상주시켜 페이지 부재 및 교체 현상을 최소화
- 기억 장치 사용 안정화 가능
![](https://velog.velcdn.com/images/kik328288/post/affd1b67-5f5f-4a63-8fb7-b7ed5d516b92/image.png)


| 장점                                               | 단점                                    |
| ------------------------------------------------ | ------------------------------------- |
| 멀티프로그래밍 정도 향상 가능 (Page Hit 증가)<br>CPU 활용률 최적화 가능 | 워킹 세트 추적 관리 복잡<br>워크 세트 크기 설정의 모호함 발생 |
## 페이지 부재 빈도(PFF: Page-Falut Frequency)
- 페이지 부재율의 상한과 하한을 정해서 직접적으로 페이지 부재율을 예측하고 조절하는 기법
- 페이지 부재 비율에 따라 페이지 프레임 개수를 조절
![](https://velog.velcdn.com/images/kik328288/post/53dcac5f-51f8-4fcd-83b6-34ed03d1cf10/image.png)


| 장점                                            | 단점                                           |
| --------------------------------------------- | -------------------------------------------- |
| 페이지 부재 발생 시 실행하여 부하 적음<br>직접적으로 페이지 부재율 조절 가능 | 프로세스를 중지시키는 과정 발생<br>페이지 참조가 새로운 지역성으로 이동 가능 |

# 지역성 (Locality)
- 프로세스 실행 간 주기억장치 참조 시 일부 페이지만 집중 참조하는 성질
- 프로세스가 집중적으로 사용하는 페이지를 알아내는 방법
- 가상 기억 장치 관리의 이론적인 근거
- 스레싱을 방지하기 위한 워킹 세트 이론의 기반

| 성질     | 설명                                                                                     |
| ------ | -------------------------------------------------------------------------------------- |
| 시간 구역성 | 한 페이지를 일정 시간 동안 집중적으로 액세스<br>반복/순환(Loop), 스택(Stack), 부 프로그램(Sub Routine), 집계(Totaling) |
| 공간 구역성 | 한 페이지 참조 시 인접한 페이지들도 집중적으로 액세스<br>배열순회(Array Traversal), 순차적 코드 실행, 같은 영역에 있는 변수 참조    |
| 순차 지역성 | 데이터가 순차적으로 액세스 되는 현상<br>프로그램 내의 명령어가 순차적으로 구성된 특성<br>EX) 순차적 코드 실행                     |
# 메모리 관리
- 한정된 메모리 공간을 효율적으로 사용하기 위해 보조 기억 장치의 프로그램이나 데이터를 주기억장치에 적재시키는 시기, 적재 위치 등을 지정하여 관리하는 기법
## 메모리 관리 기법

| 기법    | 설명                                                                           | 세부 기법                                                   |
| ----- | ---------------------------------------------------------------------------- | ------------------------------------------------------- |
| 반입 기법 | 주기억장치에 적재할 다음 프로세스의 반입 시기를 결정하는 기법<br>메모리로 적재 시기 결정 (When)                   | 요구 반입 기법<br>예상 반입 기법                                    |
| 배치 기법 | 디스크에 있는 프로세스를 주기억 장치의 어느 위치에 저장할 것인지 결정하는 기법<br>메모리 적재 위치 결정 (Where)         | 최초 적합(First Fit)<br>최적 적합(Best Fit)<br>최악 적합(Worst Fit) |
| 할당 기법 | 실행해야 할 프로세스를 주기억 장치에 어떤 방법으로 할당할 것인지 결정하는 기법<br>메모리 적재 방법 결정 (How)           | 연속 할당 기법<br>분산 할당 기법                                    |
| 교체 기법 | 재배치 기법<br>주기억 장치에 있는 프로세스 중 어떤 프로세스를 제거할 것 인지를 결정하는 기법<br>메모리 교체 대상 결정 (Who) | 프로세스의 Swap In/Out<br>FIFO, LRU, LFU                     |

### 메모리 반입 기법
- 보조 기억 장치에 보관 중인 프로그램이나 데이터를 언제 주기억장치로 적재할 것인지를 결정하는 기법

| 기법       | 설명                                                         |
| -------- | ---------------------------------------------------------- |
| 요구 반입 기법 | 다음에 실행될 프로세스가 참조 요구가 있을 경우에 적재하는 기법                        |
| 예상 반입 기법 | 시스템의 요구를 예측하여 미리 메모리에 적재하는 방법<br>요구되는 페이지 이외 다른 페이지도 함께 적재 |
### 메모리 배치 기법

| 기법                   | 설명                                                     |
| -------------------- | ------------------------------------------------------ |
| 최초 적합<br>(First Fit) | 프로세스가 적재될 수 있는 가용 공간 중에서 첫 번째 분할에 할당하는 방식              |
| 최적 적합<br>(Best Fit)  | 가용 공간 중에서 가장 크기가 비슷한 공간을 선택헤 프로세스 적재하는 방식<br>공백 최소화 장점 |
| 최악 적합<br>(Worst Fit) | 프로세스의 가용 공간 중에서 가장 큰 공간에 할당하는 방식                       |
### 메모리 할당 기법
- 프로세스를 실행시키기 위해 주기억장치에 어떻게 할당할 것인지에 대한 기법

| 종류       | 설명                                                                                                                                              |
| -------- | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| 연속 할당 기법 | 실행을 위한 각 프로세스를 주기억장치 공간 내에서 인접되게 연속하여 저장하는 방법<br>프로세스를 주기억장치에 연속으로 할당하는 기법<br>단일 분할 할당 기법 : 오버레이, 스와핑<br>다중 분할 할당 기법 : 고정 분할 할당 기법, 동적 분할 할당 기법 |
| 분산 할당 기법 | 하나의 프로세스를 여러 개의 조각으로 나누어 주기억장치 공간 내에 분산하여 배치하는 기법<br>주로 가상 기억 장치에서 사용<br>종류 : 페이징 기법, 세그먼테이션 기법, 페이징/세그먼테이션 기법                                  |

#### Segmentation 기법
- 가상 기억 장치에 보관된 프로그램을 ==다양한 크기의 논리적 단위==로 분할 후 (Segment) 주기억장치에 배치 및 실행
- 프로그램을 논리적 단위의 가변 크기 블록으로 나누는 방식 
- 외부 단편화 발생 가능성 존재
- Paging 기법의 경우, 프로그램을 동일 일정 크기로 분할함

| 장점                                   | 단점           |

| 가변적인 데이터 구조와 모듈 처리<br>자원의 효율적인 이용 가능 | 외부 단편화 현상 발생 |


#### Paging 기법
- 가상 기억 장치를 고정된 크기의 페이지 단위로 나누는 방식
- 프로그램을 동일 일정 크기로 분할함
- 각 페이지는 물리 메모리의 프레임과 1:1 매핑
- 내부 단편화 발생 가능성 존재

| 장점                                     | 단점                                             |
| -------------------------------------- | ---------------------------------------------- |
| 공유 페이지 이용<br>메모리 활용을 통한 다중 처리 프로그래밍 가능 | 페이징 사상을 위한 하드웨어 비용 소요<br>속도 저하<br>내부 단편화 현상 발생 |

|      항목      | 페이지 크기 작을 경우 | 페이지 크기가 클 경우 |
| :----------: | :----------: | :----------: |
| 페이지 맵 테이블 크기 |      큼       |      작음      |
|    내부 단편화    |      작음      |      큼       |
|   입/출력 시간    |    오래 걸림     |      짧음      |
| 기억 장소 이용 효율  |      높음      |      낮음      |

- 외부 단편화 및 내부 단편화 최소화를 위해 개발
# 메모리 단편화
- 분할된 주기억장치에 프로세스를 할당, 반납 과정에서 사용되지 못하고 낭비되는 기억 장치가 발생하는 현상
## 내부 단편화 (Internal Fragmentation)
- 분할된 공간에 프로세스를 적재한 후 남은 공간
- 고정 분할 할당 방식 또는 페이징 기법 사용 시 발생하는 메모리 단편화
![](https://velog.velcdn.com/images/kik328288/post/a9c6c465-0784-4a22-bb99-009f17b59662/image.png)

### 내부 단편화 해결 방안

|           해결 방안            | 설명                                                                                      |
| :------------------------: | --------------------------------------------------------------------------------------- |
| 슬랩 할당자<br>(Slab Allocator) | 동적 메모리 관리 기법<br>1. 페이지 프레임을 할당 받아 공간을 작은 크기로 분할(캐시 집합)<br>2. 메모리 요청 시 작은 크기로 메모리를 할당/해제 |
|     통합<br>(Coalescing)     | 인접한 단편화 영역을 찾아 하나로 통합하는 기법                                                              |
|     압축<br>(Compaction)     | 메모리의 모든 단편화 영역을 하나로 압축하는 기법                                                             |
## 외부 단편화 (External Fragmentation)
- 할당된 크기가 프로세스 크기보다 작아서 사용하지 못하는 공간
- 가변 분할 할당 방식 또는 세그먼테이션 기법 사용 시 발생하는 메모리 단편화
![](https://velog.velcdn.com/images/kik328288/post/f9fae439-437e-4b3f-96e9-2ec627285e5e/image.png)

### 외부 단편화 해결 방안

|                 해결 방안                  | 설명                                                                |
| :------------------------------------: | ----------------------------------------------------------------- |
| 버디 메모리 할당<br>(Buddy Memory Allocation) | 요청한 프로세스 크기에 가장 알맞은 크기를 할당하기 위해 메모리를 $2^n$의 크기로 분할하여 메모리를 할당하는 기법 |
|           통합<br>(Coalescing)           | 인접한 단편화 영역을 찾아 하나로 통합하는 기법                                        |
|           압축<br>(Compaction)           | 메모리의 모든 단편화 영역을 하나로 압축하는 기법                                       |

3. 개발자들에게 필요한 기능들을 추상화하고, 객체, 클래스 등을 제공한다.
## 분산 처리 시스템의 투명성(Transparency)
- 사용자가 분산된 여러 자원의 위치 정보를 알지 못하고 마치 하나의 커다란 시스템을 사용하는 것처럼 인식하도록 하는 것
### 투명성 종류
1. 위치 투명성 (Location)
	- 하드웨어와 소프트웨어의 물리적 위치를 사용자가 알 필요가 없음
	- 분산 운영체제에서 사용자가 원하는 파일이나 데이터베이스, 프린터 등의 자원들이 지역 컴퓨터 또는 네트워크 내의 다른 원격지 컴퓨터에 존재하더라도 위치에 관계 없이 그 외 사용을 보장하는 개념
2. 이주 투명성 (Migration)
	- 사용자나 응용 프로그램의 동작에 영향을 받지 않고 시스템 내에 있는 정보 객체를 이동할 수 있게 함
3. 복제 투명성 (Replication)
	- 사용자에게 통지할 필요 없이 시스템 안에 파일들과 자원들의 부가적인 복사가 자유로움
4. 병행 투명성 (Concurrency)
	- 다중 사용자들이 자원들을 자동으로 공유할 수 있음
