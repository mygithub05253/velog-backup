# SW 개발 보안
- 소스 코드 등에 존재하는 보안 취약점을 제거하고, 보안을 고려하여 기능을 설계 및 구현하는 등 소프트웨어 개발 과정에서 지켜야할 일련의 보안 활동

|      위험<br>(Risk)      | 위협이 취약점을 이용해 조직의 자산 손실 피해를 가져올 가능성                                            |

- 불필요한 서비스 및 악성 프로그램의 확인과 제거
- 실행 중인 프로세스 및 열린 포트 위주로 확인

# 보안 취약성
- 정보 시스템에 불법적인 사용자의 접근, 정상적인 서비스를 방해, 정보 시스템에서 관리하는 중요 데이터의 유출/변조/삭제에 대한 위협
- 위협이 발생하기 위한 사전 조건으로 시스템의 정보 보증을 낮추는 데 사용되는 약점
- 응용 프로그램의 보안 취약점, 위협 요소, 허점을 탐지하는 데 도움
## 취약성 공격
- 공격자가 원하는 결과를 얻기 위해서 악성 코드/악성 프로그램 등을 이용해 보안 버그나 잘못된 설정 등의 결함에 대한 공격
- 목표 : 대상 시스템을 장악하여 정보를 빼내거나 혹은 다른 용도로 악용하는 것
- 피해 시스템의 중요 데이터를 암호화하여 인질로 삼고 금전을 요구하는 랜섬웨어 역시 취약점 공격을 통해 권한 획득 중
### 취약성에 따른 공격 기법

|                기법                 | 설명                                                                                                                 |
| :-------------------------------: | ------------------------------------------------------------------------------------------------------------------ |
|        익스플로잇<br>(Exploit)         | 컴퓨터의 소프트웨어나 하드웨어 및 컴퓨터 관련 전자 제품의 버그/보안 취약점 등 설계상 결함을 이용해 공격자의 의도된 동작을 수행하도록 만들어진 절차나 일련의 명령, 스크립트, 프로그램을 사용한 공격 행위 |
|     코드 실행<br>(Code Execusion)     | 응용 프로그램이 적절한 입력 유효성 검사 없이 쉘(Shell) 명령을 실행하는 취약점을 이용하여 공격자가 원하는 임의 코드가 실행되도록 하는 공격 기법                               |
|   버퍼 오버플로우<br>(Buffer Overflow)   | 정해진 메모리의 범위를 넘치게 해서 원래의 리턴 주소를 변경시켜 임의의 프로그램이나 함수를 실행시키는 시스템 해킹 기법                                                 |
| 디렉터리 접근<br>(Directory Traversal)  | HTTP 기반의 공격으로 액세스가 제한된 디렉터리에 접근하여, 서버의 루트 디렉터리에서 외부 명령을 실행하여 파일, 웹 소스 등을 강제로 내려받을 수 있는 공격 기법                       |
| FTP 바운스 공격<br>(FTP Bounce Attack) | FTP 서버가 데이터를 전송할 때 목적지가 어디인지 검사하지 않는 설계 상의 문제점을 이용한 공격 기법                                                          |

# 보안 인증 방법
1. Something You Know (알고 있는 것)
	- 사용자가 알고 있는 정보를 사용해 인증하는 방법
	- 패스워드나 PIN 같은 비밀 정보를 사용해 인증
2. Somthing You Have (가지고 있는 것)
	- 사용자가 소유하고 있는 물리적인 장치나 객체를 사용해 인증하는 방법
	- 스마트 카드, USB 토큰, 휴대폰 앱 등을 사용해 인증
3. Something You Are (자신의 특징)
	- 사용자의 생체적인 특징이나 생체 인식 기술을 사용해 인증하는 방법
	- 지문, 홍채, 음성, 얼굴 등 개인의 생체 특징을 사용해 인증하는 바이오메트릭 인증
4. Somwhere You Are (있는 곳)
	- 사용자가 특정한 위치에 있는 지를 확인해 인증하는 방법
	- IP 주소나 지리적 위치를 이용해 인증
# 소프트웨어 정의 기술
- SDE : Software-defined Everything
## SDDC (Software-Defined Data Center)
- 소프트웨어 정의 데이터 센터
- 모든 하드웨어가 가상화되어 가상의 자원 풀을 구성
- 데이터 센터 전체를 운영하는 소프트웨어가 필요한 기능 및 규모에 따라 동적으로 자원을 할당, 관리하는 역할을 수행하는 데이터 센터
- 데이터 센터를 가상화하여 인력 개입 없이 SW 만으로 관리/제어 가능한 데이터 센터
### SDDC 특징
1. 컴퓨팅, 네트워킹, 스토리지, 관리 등을 모두 소프트웨어로 정의
2. 인력 개입 없이 소프트웨어 조작만으로 자동 제어 관리
3. 데이터 센터 내 모든 자원을 가상화하여 서비스
### SDDC 구성 요소

| 요소                      | 설명                                                                                                                                                                                                                                                |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| SDC                     | Software Defined Computing<br>소프트웨어 정의 컴퓨팅 환경<br>서버의 CPU, 메모리에 대해서 소프트웨어 명령어 기반으로 제어 가능한 컴퓨터                                                                                                                                                      |
| SDN                     | Software Defined Networking => 소프트웨어 정의 네트워킹<br>개방형 API를 통해 네트워크의 트래픽 전달 동작을 소프트웨어 기반 컨트롤러에서 제어/관리하는 가상화 네트워크 기술<br>네트워크를 제어부/데이터 전달부로 분리해서 보다 효율적으로 네트워크 제어/관리<br>기존 네트워크에 영향을 주지 않으면서 특정 서비스의 전송 경로 수정을 통해 인터넷 문제 처리                            |
| SDS                     | Software Defined Storage => 소프트웨어 정의 스토리지<br>서버와 전통적인 스토리지 장치에 장착된 물리적 디스크 드라이브를 가상화 기술을 적용하여 필요한 공간만큼 나눠서 사용할 수 있도록 논리적인 스토리지로 통합한 가상화 기술<br>컴퓨팅 소프트웨어로 규정하는 데이터 스토리지 체계<br>일정 조직 내 여러 스토리지를 하나의 스토리지처럼 관리하고 운용하는 컴퓨터 이용 환경 => 이를 반대로 수행하는 것도 가능 |
| 프로비저닝<br>(Provisioning) | SDDC 자원에 대한 할당 관리 기술                                                                                                                                                                                                                              |


# 소프트웨어 신기술 (네트워크 관련)
## CPS (Cyber-Physical System)
- 가상 물리 시스템
- 인간의 개입 없이 대규모 센서/액추에이터를 갖는 물리적인 요소들과 통신 기술, 응용/시스템 소프트웨어 기술을 활용하여 실시간으로 물리적 요소들을 제어하는 컴퓨팅 요소가 결합된 복합 시스템
### 액추에이터 (Actuator)
- CPS에서 센서로부터 수집된 데이터나 시스템의 명령을 받아 기계적인 움직임 등의 물리적인 동작을 수행하는 장치
## 디지털 트윈 (Digital Twin)
- 물리적인 사물과 컴퓨터에 동일하게 표현되는 가상 모델
- 실제 물리적인 자산 대신 소프트웨어로 가상화함으로써 실제 자산의 특성에 대한 정확한 정보 획득 가능
- 자산 최적화, 돌발 사고 최소화, 생산성 증가 등 설계부터 제조, 서비스에 이르는 모든 과정의 효율성 향상 가능
- 물리적 사물을 가상화하여(twin) 실제 자산의 특성 정보를 구현
## 서비스 지향 아키텍처 (SOA : Service Oriented Architecture)
- 서비스라고 정의되는 분할된 애플리케이션 조각들을 느슨한 결합을 통해 하나의 완성된 애플리케이션을 구현하기 위한 아키텍처
- 구성 계층 : 표현 / 업무 프로세스 / 서비스 중간 / 애플리케이션 / 데이터 저장
- SOA는 비즈니스/표현/프로세스 층으로 구성
## 마이크로서비스 아키텍처 (MSA : Micro Services Architecture)
- 하나의 큰 시스템을 여러 개의 작은 서비스로 나누어 변경과 조합이 가능하도록 만든 아키텍처
## 핀테크 (Fintech)
- 금융과 기술의 합성어 (Financial + Technology)
- 모바일, 소셜 네트워크, 빅데이터 등의 첨단 기술을 활용한 새로운 형태의 금융 기술
## 디지털 변혁 (Digital Transformation)
- 디지털 기술 기반으로 기업의 전략, 조직, 프로세스, 비즈니스 모델, 문화, 커뮤니케이션 등을 변화시키는 경영 전략
## RPA (Robotic Process Automation)
- 사용자가 PC 및 모바일 화면에서 반복 수행하는 정형화된 업무를 Bot이 대신 수행하여 자동화하는 솔루션
## 양자암호통신 (Quantum Cryptogrphy and Communications)
- 양자 중첩, 얽힘, 불확정성의 원리 등 양자 역학의 원리를 기반으로 암호용 키를 송/수신부에 분배하고 암호화 통신을 진행하는 양자 역학 기반 보안 통신 기술
## 양자 키 분배 (Quantum Key Distribution)
- 양자 통신을 위해 비밀키를 분배하여 관리하는 기술
- 키 분배를 위해 얽힘 상태 광자 또는 단일 광자 이용
## 매시업 (Mashup)
- 웹으로 제공하고 있는 정보와 서비스를 융합하여 새로운 소프트웨어나 서비스, 데이터베이스 등을 만드는 기술
- 서로 다른 웹 사이트의 컨텐츠를 조합하여 새로운 차원의 컨텐츠나 서비스를 창출하는 웹 사이트 또는 애플리케이션
- 웹에서 제공하는 정보/서비스를 이용하여 새로운 SW 제작
## 그레이웨어 (Grayware)
- 바이러스나 악성 코드를 포함하지 않는 합법적 프로그램
- 사용자를 귀찮게 하거나 위험한 상황에 빠드릴 수 있는 프로그램
- 평범한 소프트웨어인지, 바이러스인지 구분하기 어려운 중간 영역에 존재하는 프로그램
- 스파이웨어, 애드웨어, 장난 프로그램, 원격 액세스 도구 등 사용자가 원하지 ㅇ낳는 프로그램을 총칭하는 이름
### 스파이웨어 (Spyware)
- 사용자의 동의 없이 컴퓨터에 몰래 설치됨
- 중요한 개인 정보 등을 수집하는 악의적인 컴퓨터 소프트웨어
### 애드웨어 (Adware)
- 사용 기간의 제한이 없이 무료 사용과 배포 가능
- 광고 시청을 전제로 사용이 허용되는 프로그램
## 텐서플로 (TensorFlow)
- 구글의 구글 브레인 팀이 제작하여 공개한 기계학습을 위한 오픈 소스 소프트웨어 라이브러리
## 포스퀘어 (Foursquare)
- 위치 기반 소셜 네트워크 서비스이자 이를 개발한 회사의 명칭으로 사용자는 포스퀘어 서비스에서 자신의 휴대 전화의 애플리케이션 또는 단문 메시지 서비스를 이용해 특정 장소에 “체크인” 가능한 기술
## 파스타 (PaaS-TA)
- 인프라 제어 및 관리/실행/개발/서비스/운영 환경으로 구성
- NIA 주도로 국내 IT 서비스 경쟁력 강화 목적으로 개발된 개방형 클라우드 컴퓨팅 플랫폼
- 국내 IT 서비스 경쟁력 강화를 목표로 개발되었으며 인프라 제어 및 관리 환경, 실행 환경, 개발 환경, 서비스 환경, 운영 환경으로 구성되어 있는 개방형 클라우드 컴퓨팅 플랫폼
- 클라우드 상에서 애플리케이션을 쉽게 개발/운영할 수 있도록 지원
- 공공기관 및 기업에서 클라우드 전환/디지털 전환에 활용 가능
## 메타버스
- 가상/초월과 세계/우주의 합성어
- 3차원 가상 세계를 뜻하는 용어
- 정치/경제/사회/문화의 전반적인 측면에서 현실과 비현실 모두 공존할 수 있는 생활형/게임형 가상 세계
## 가상 현실 (VR: Virtual Reality)
- 사용자를 가상 세계로 몰입시키는 인터랙티브한 경험을 제공하는 기술
## 증강 현실 (AR: Augmented Reality)
- 가상 현실(VR)의 한 분야
- 실제로 존재하는 환경에 가상의 사물이나 정보를 합성하여 마치 원래의 환경에 존재하는 사물처럼 보이도록 하는 기술
## 혼합 현실 (MR : Mixed Reality)
- 실세계의 물리적 환경과 가상 환경을 혼합한 경험을 제공하는 기술
## GPUaaS (GPU as a Service)
- 클라우드를 통해 GPU 자원을 필요할 때마다 유연하게 사용할 수 있는 서비스
- 사용자가 직접 고가의 GPU 하드웨어를 구매하거나 유지 관리하지 않고도 인터넷을 통해 GPU 자원을 빌려서 사용할 수 있는 서비스
- 대규모 데이터 세트 학습 시 병렬 연산 수행 및 그래픽 렌더링 처리 시 활용

# 암호 알고리즘 (Encryption Algorithm)
- 데이터의 무결성 및 기밀성 확보를 위해 정보를 쉽게 해독할 수 없는 형태로 변환하는 기법
## 암호 알고리즘 주요 용어

| 용어    | 설명                                                                        |
| ----- | ------------------------------------------------------------------------- |
| 평문    | Plain/Plaintext<br>암호화 되기 전의 원본 메시지                                       |
| 암호문   | Cipher/Ciphertext<br>암호화가 적용된 메시지                                         |
| 암호화   | Encrypt/Encryption/Encoding<br>평문을 암호문으로 바꾸는 작업                           |

| 치환 암호 | 대치 암호, Substitution Cipher<br>비트, 문자 또는 문자의 블록을 다른 비트, 문자 또는 블록으로 대체하는 방법 |
| 전치 암호 | Transposition Cipher<br>비트, 문자 또는 블록이 원래 의미를 감추도록 자리바꿈 등을 이용해 재배열         |

### 암호화 방식
#### 양방향 암호화 방식
![](https://velog.velcdn.com/images/kik328288/post/38ad40fa-9e44-4155-aa3e-08f1dbbc7871/image.png)

- DES : 미국 NBS에서 발표한 개인키 알고리즘
	- 블록 크기 ==64비트== / 키 길이 56 비트
- AES : 미국 표준 기술 연구소 (NIST) 개인키 알고리즘
	- 블록 크기 128/192/256 비트
- RSA : MIT 공개키 암호화 알고리즘
	- ==소인수분해 어려운 큰 소수 숫자 활용==
- 라빈(Rabin) 암호
	- RSA 암호 시스템의 변형
	- 소인수분해를 기반으로 하는 간단하고 빠른 연산 속도의 공개키 암호 방식
- ECC 암호 (타원 곡선 암호)
	- RSA 암호보다 짧은 키 길이로 동일한 보안 수준 제공
	- 특허 문제 존재
- 엘가말 (El Gamal) 암호
	- RSA 암호와 유사
	- 암호화 및 복호화 속도가 느림
![](https://velog.velcdn.com/images/kik328288/post/6aef432f-d44b-4dc6-a0a3-f79f3860bba7/image.png)

##### 대칭 키 암호 방식 및 종류
- 비교적 실행 속도가 빠르기 때문에 다양한 암호의 핵심 함수로 사용 가능
- 반드시 비밀키를 상대방에게 전달해야 하며, 이 과정이 보안상 가장 취약
- 속도는 빠르지만 키 교환이 필요
![](https://velog.velcdn.com/images/kik328288/post/ae93cce5-de1c-450b-a125-858d61d24656/image.png)

##### 비대칭 키 암호 방식
- 자신만이 보관하는 비밀키를 이용해 인증, 전자서명 등에 적용 가능
![](https://velog.velcdn.com/images/kik328288/post/5f9bfdac-73a1-4542-881e-80069b5f6b39/image.png)

#### 단방향 암호화 방식 - 해쉬 (Hash)
- 해쉬 알고리즘 (해쉬 함수)로 불리며, 해쉬 함수로 변환된 값/키를 해쉬값/키라 함
- 임의의 길이의 입력 데이터를 받아 고정된 길이의 해쉬 값을 변환
- 해시 암호화 알고리즘이 적용된 정보 복호화 불가능
- 종류 : SHA 시리즈 / HAVAL / MD4 / MD5 / N-NASH / SNEFRU

## 암호 공격 유형

| 유형        | 설명                                                                                                |
| --------- | ------------------------------------------------------------------------------------------------- |
| 암호문 단독 공격 | COA : Ciphertext Only Attack<br>공격자가 암호문만을 가지고, 암호 시스템을 분석하거나 평문을 추측하는 공격 기법                      |
| 알려진 평문 공격 | KPA : Known Plaintext Attack<br>공격자가 일부 암호문-평문 쌍을 가지고 있을 때, 암호 시스템을 해독하려는 공격 기법                   |
| 선택 평문 공격  | CPA : Chosen Plaintext Attack<br>공격자가 자유롭게 선택한 평문을 암호화하여 해당 암호문을 얻을 수 있는 상황에서, 암호 시스템을 공격하는 기법    |
| 선택 암호문 공격 | CCA : Chosen Ciphertext Attack<br>공격자가 자유롭게 선택한 암호문을 복호화하여 해당하는 평문을 얻을 수 있는 상황에서, 암호 시스템을 공격하는 기법 |

## 상향식 비용 산정 기법
- 프로젝트 세부 작업 단위 별로 비용 정산 후 전체 비용을 산정하는 방법
### 상향식 비용 산정 기법 종류
#### LoC (Lines of Code)
- 소프트웨어 각 기능의 원시 코드 라인 수의 낙관치, 중간치, 비관치를 측정하여 예측치를 구하고 이를 이용해 비용을 산정
	- 낙관치 : 가장 적게 측정된 코드 라인 수
	- 중간치 : 측정된 모든 코드 라인 수의 평균
	- 비관치 : 가장 많이 측정된 코드 라인 수
- 측정이 쉽고 이해하기 쉬움
- 예측치를 이용해 생산성, 노력 개발 기간 등의 비용 산정
![](https://velog.velcdn.com/images/kik328288/post/9deb4c2e-ff3d-4696-bc73-beffc6a5b0cb/image.png)

#### Man Month
- 한 사람이 1개월 동안 할 수 있는 일의 양을 기준으로 프로젝트 비용을 산정하는 기법
- 공식
$$ Man\ Month = (LoC)/(프로그래머의\ 월간\ 생산성) $$
$$ 프로젝트\ 기간= (Man\ Month)/(프로젝트\ 인력) $$
#### 개발 단계 노력 기법
- LoC 기법을 확장한 기법
- LoC 기법을 코딩 단계 뿐만 아니라 SW 개발 생명 주기 단계별로 적용시켜 모든 단계에서의 비용ㅇㄹ 산정하는 기법
### 수학적 비용 산정 기법
#### COCOMO (Constructive Cost Model)
- 보헴 (Bohem)이 제안
- 원시 코드 라인 수 기반 비용 산정 기법
- 비용 견적 강도 분석 및 비용 견적의 유연성이 높아 소프트웨어 개발비 견적에 널리 통용
- 같은 프로젝트라도 성격에 따라 비용이 다르게 산정
- 산정 결과는 프로젝트를 완성하는데 필요한 man-month로 표현

|           유형            | 설명                                                                    |
| :---------------------: | --------------------------------------------------------------------- |
|    조직형<br>(Organic)     | 일괄 자료 처리나 과학 기술 계산용, 비즈니스 자료 처리용<br>중/소규모 SW용<br>==5만 라인(50KDSI) 이하== |
| 반분리형<br>(Semi-detached) | ==30만 라인 (300KDSI) 이하==의 트랜잭션 처리 시스템, 운영체제, 데이터베이스 관리 시스템 등           |
|    내장형<br>(Embedded)    | ==30만 라인 (300KDSI) 이상==의 최대 규모 SW 관리                                  |
#### PUTNAM
- SW 생명 주기 전 과정에 사용될 ==노력의 분포==를 이용한 비용 산정
- Rayleigh Norden 곡선의 노력 분포도를 기초로 함
- ==SLIM== : Rayleigh-Norden 곡선 / Putnam 모형 기초로 개발된 자동화 추정 도구
#### Function Point (FP)
- SW 기능 증대 요인에 가중치 부여 후 합산하여 기능 점수 산출
- 요구 기능을 증가시키는 인자별로 가중치를 부여하고, 요인별 가중치를 합산하여 총 기능의 점수를 계산하여 비용을 산정하는 방식
$$ 기능점수(FP)=총\ 기능점수 * [0.65+(0.1*총\ 영향도)] $$
- 경험을 바탕으로 단순, 보통, 복잡한 정도에 따라 가중치를 부여
##### SW 기능 증대 요인
1. 자료 입력 (입력 양식)
2. 정보 출력 (출력 보고서)
3. 명령어 (사용자 질의 수)
4. 데이터 파일
5. 인터페이스

|          Smurfing <br>(스머핑)           | ==IP / ICMP 특성==을 악용해 엄청난 양의 데이터를 한 사이트에 집중적으로 보내 네트워크를 불능 상태로 변경                                                                                                                                       |

|           Switching Jamming           | 위조된 매체 접근 제어(MAC) 주소를 지속적으로 보내 스위치 MAC 주소를 혼란시켜 더미 허브 (Dummy Hub)처럼 작동                                                                                                                                  |
## Smurf Attack
- 공격자는 브로드캐스트 주소를 이용해 ICMP Echo 요청을 다수의 호스트에 전송
- 이후 응답이 모두 피해자 IP로 향하게 하여 대량의 ICMP 패킷이 특정 대상에게 집중되도록 하는 방식으로 서비스 거부 공격 유발
- ICMP 프로토콜의 특성을 악용한 증폭형 공격

### DDoS 공격 대응 방안
1. 차단 정책 업데이트 : 공격 규모를 확인하여 가용성이 침해될 수 있는 지점을 확인 및 데이터 기반 차단 정책 업데이트
2. 좀비 PC IP 확보 : 공격자는 대부분 Source IP를 위조하므로 IP 위변조 여부를 확인하는 절차 필요
3. 보안 솔루션 운영 : 방화벽, 침입 탐지 시스템 등의 보안 솔루션 운영
4. 홈페이지 보안 관리 : 홈페이지에 대한 모의해킹 등을 수행하여 보안 유지
5. 시스템 패치 : 시스템에 존재하는 취약점을 패치를 통해 해결
## DRDoS 공격 (Distributed Reflection DoS)
- 공격자가 출발지 IP를 공격 대상 IP로 위조하여 다수의 반사 서버로 요청 정보를 전송하고 공격 대상자는 반사 서버로부터 다량의 응답을 받아서 서비스 거부가 되는 공격
### DRDoS 공격 방식

### DRDoS 공격 절차
1. 출발지 IP 변조 : 공격자는 출발지 IP를 공격 대상자 IP로 Spoofing하여 SYN 패킷을 공격 경유지 서버로 전송
2. 공격 대상자 서버로 응답 거부 : SYN 패킷을 받은 경유지 서버는 Spoofing 된 IP(공격 대상자 서버)로 SYN/ACK 전송
3. 서비스 거부 : 공격 대상자 서버는 수많은 SYN/ACK를 받게 되어 서비스 거부가 됨
### DRDoS 대응 방안
1. ISP(인터넷 서비스 사업자)가 직접 차단
2. 반사 서버에서 연결을 완료하지 않은 Syn 출처 IP를 조사하여 블랙 리스트로 운용, 공격 서버를 사전에 차단
3. 공격 대상이 되고 있는 공격 대상 서버 IP와 Port(서비스)를 변경, 필터링하여 운영

### 침입 탐지 시스템 (IDS : Intrusion Detection System)
1. 오용 탐지 기법 (Misuse Detection)
	- Signature Base나 Knwoledge Base
	- 이미 발견되고 정립된 공격 패턴을 입력해두었다가 탐지 및 차단
2. 이상 탐지 기법 (Anomaly Detection)
	- 평균적인 시스템의 상태를 기준으로 비정상적인 행위나 자원의 사용이 감지되면 이를 알려주는 시스템
3. HIDS (Host-Based Intrusion Detection)
	- 운영체제에 설정된 사용자 계정에 따라 어떤 사용자가 접근을 시도하고 어떤 작업을 했는지에 대한 기록을 남기고 추적
4. NIDS (Network-Based Intrusion Detection System)
	- 대표적으로 Snort 존재
5. DMZ (Demilitarized Zone)
	- 외부 인터넷에 서비스를 제공하는 서버가 위치하는 네트워크
	- IDS 설치 가능
# 서버 장비 운영
## 저장 장치 (스토리지 시스템)
- 정보 시스템 구축 및 운영을 위해서는 많은 저장 공간의 확보가 필요하며 이런 대용량의 데이터를 저장하기 위한 장치
### DAS (Direct Attached Storage)
- 하드 디스크와 같은 데이터 저장 장치를 호스트 버스 어댑터에 직접 연결하는 방식으로 구성
- 저장 장치와 호스트 기기 사이에는 네트워크 디바이스가 있지 않아야 하고 직접 연결하는 방식으로 구성된 기술

|     | 설명                                               |
| --- | ------------------------------------------------ |
| 장점  | 주어진 데이터 전송 성능을 보장<br>전용 케이블을 사용해 안전성이 우수         |
| 단점  | 파일 시스템의 공유가 어려움<br>확장성/유연성 부족<br>연결되는 서버의 수가 제한적 |
### NAS (Network Attached Storage)
- 서버와 저장 장치를 네트워크로 연결하는 방식으로 구성된 기술
- 구성 설정이 간편하여 별도의 운영체제를 가진 서버 한 곳에서 파일을 관리하기 때문에 서버 간에 스토리지 및 파일 공유 용이
- 저장 장치와 서버를 직접 연결하는 것이 아니라 네트워크를 통해 스토리지에 접속하고, 파일 단위로 관리

|     | 설명                                                              |
| --- | --------------------------------------------------------------- |
| 장점  | 이기종 간 파일 공유가 가능<br>설치 및 관리 용이                                   |
| 단점  | 네트워크 대역폭 잠식 가능<br>OLTP 성능 저하 문제 발생 가능<br>채널 접속 단계 증가로 장애 포인트 증가 |
### SAN (Storage Area Network)
- 서버와 스토리지를 저장 장치 전용 네트워크로 상호 구성하여 고가용성, 고성능, 융통성, 확장성을 보장하고 데이터를 블록(Block) 단위로 관리하는 기술
- DAS의 빠른 처리와 NAS의 스토리지 공유 방식의 장점을 합친 방식
- 광케이블 및 광 채널 스위치를 통해 근거리 네트워크 환경을 구성하여 빠른 속도로 데이터를 처리할 수 있으며, 스토리지 공유가 가능한 기술
- 네트워크 상에 광 채널 스위치의 이점인 고속 전송과 장거리 연결 및 멀티 프로토콜 기능 활용 가능

|     | 설명                                                                                                           |
| --- | ------------------------------------------------------------------------------------------------------------ |
| 장점  | 각기 다른 운영체제를 가진 여러 기종이 네트워크 상에서 동일 저장 장치의 데이터를 공유 → 여러 개의 저장 장치나 백업 장비 단일화 가능<br>무정지 확장성, 고성능, 관리 효율성이 높음<br> |
| 단점  | 비용이 높음                                                                                                       |

## 백업 (Back-up)
- 조직의 중요한 자산인 데이터를 예기치 않은 사고로 유발된 원본 데이터의 손상 및 분실로부터 보호하기 위해 특정 시점 기준으로 사본을 사전에 생성하는 절차 및 작업
### 백업 유형

|               유형                | 설명                                  |
| :-----------------------------: | ----------------------------------- |
|     전체 백업<br>(Full Backup)      | 백업 받고자 하는 데이터 전체에 대해 백업하는 방식        |
| 차등 백업<br>(Differential Backups) | 마지막 전체 백업 이후 변경된 모든 데이터를 백업하는 방식    |
|  증분 백업<br>(incremental Backup)  | 정해진 시간을 기준으로 그 이후에 변경된 파일만을 백업하는 방식 |
## 고가용성 기술(HA : High Availability)
- 두 개 이상의 시스템을 클러스터로 구성하여 하나의 시스템이 장애 시 최소한의 서비스 중단을 위해 다른 시스템으로 신속하게 시스템을 대체 작동시키는 기술
- 잦은 DDoS 공격이나 장애 발생 시, 두 대 이상의 서버를 구성하여 하나의 서버가 다운되면 다른 서버가 자동으로 서비스를 이어받아 중단 없이 운영될 수 있도록 하는 방식
- 목표
	- 기업의 서비스 고장 시간 최소화로 가용성을 극대화
	- 비즈니스 연속성을 확보하여 수익성을 최대화
### 고가용성 솔루션 (HACMP)
- High Availability Cluster Multi-Processing
- 2대 이상의 시스템을 1개의 클러스터로 묶어서 각 시스템을 모니터링하고, 메인 시스템의 장애 발생 시 서브 시스템으로 전환하여 시스템 중단을 최소화해주는 기술
- 조직, 기업의 기간 업무 서버 등의 안정성을 높이기 위해 사용
- 각 시스템 간에 공유 디스크를 중심으로 클러스터링으로 엮어 다수의 시스템을 동시에 연결 가능
- 여러 가지 방식으로 구현되며 2개의 서버를 연결하는 것으로 2개의 시스템이 각각 업무를 수행하도록 구현하는 방식이 널리 상용

- 다중 사용자 시스템, 망 운영 시스템에서 접속자의 로그인 정보를 확인하는 보안 절차
- 전송된 메시지 무결성 및 송신자를 검증하는 과정이 인증에 해당
- 서버에서 사용자들에게 접근 권한을 부여하기 위해 접속자의 신분을 검증하는 과정을 로그인이라 하며 인증 과정 필요

# 서버 접근 통제
- 사람 또는 프로세스가 서버 내 파일에 읽기, 쓰기, 실행 등의 접근 여부를 허가하거나 거부하는 기능
## 서버 접근 통제의 목적
- 비인가자로부터 객체의 기밀성, 무결성, 가용성 보장
- 접근 통제를 위해 인증, 인가, 계정 관리 사용

|           요소           | 설명                                 |
| :--------------------: | ---------------------------------- |
| 인증<br>(Authentication) | 접근을 시도하는 가입자 또는 단말에 대한 식별 및 신분을 검증 |
| 인가<br>(Authorization)  | 검증된 가입자나 단말에게 어떤 수준의 권한과 서비스를 허용   |
| 계정 관리<br>(Accounting)  | 리소스 사용에 대한 정보를 수집하고 관리하는 서비스       |
## 서버 접근 통제 기술 (Access Control)

|                   기술                   | 설명                                                                                                                                           |
| :------------------------------------: | -------------------------------------------------------------------------------------------------------------------------------------------- |
| 임의 접근 통제<br>(DAC)<br>Discretionary AC  | ==사용자의 신원/신분에 따라== 접근 권한 부여<br>==데이터 소유자==가 접근 통제 권한 지정/제어<br>객체 생성자가 모든 권한을 갖고, 다른 사용자에게 허가<br>SQL 명령어 : GRANT / REVOKE                     |
|   강제 접근 통제<br>(MAC)<br>Mandatory AC    | ==주체와 객체의 등급을 비교 후 시스템이 접근 권한 부여==<br>DB 객체별로 보안 등급 부여 및 사용자별로 인가 등급 부여<br>자신보다 보안 높은 객체에 읽기/수정/등록 불가하나 등급 같은 객체에는 모두 가능, 자신보다 낮은 객체에는 읽기 가능 |
| 역할 기반 접근 통제<br>(RBAC)<br>Role-based AC | ==사용자의 역할에 따라== 접근 권한 부여<br>==중앙 관리자가 지정==<br>다중 프로그래밍에 최적화                                                                                  |
### MAC
- 정보 시스템 내에서 어떤 주체가 특정 개체에 접근하려 할 때 양쪽의 보안 레이블에 기초하여 높은 보안 수준으로 요구하는 정보(객체)가 낮은 보안 수준의 주체에게 노출되지 않도록 하는 접근 제어 방법
### 서버 접근 통제 간 비교

|  정책   |     DAC      |      MAC      |  RBAC  |
| :---: | :----------: | :-----------: | :----: |
| 권한 부여 |   데이터 소유자    |      시스템      | 중앙 관리자 |
| 접근 결정 |      신분      |     보안 등급     |   역할   |
| 정책 변경 |    변경 용이     |      고정적      | 변경 용이  |
|  장점   | 구현 용이<br>유연함 | 안정적<br>중앙 집중적 | 관리 용이  |
## 접근 제어 (Access Control)
- 적절한 권한을 가진 인가자만 특정 시스템이나 정보에 접근할 수 있도록 통제하는 것
- 시스템 및 네트워크에 대한 접근 제어의 가장 기본적인 수단은 IP와 서비스 포트
- 네트워크 장비에서 수행하는 IP에 대한 접근 제어로는 관리 인터페이스의 접근 제어와 ACL (Access Control List) 등이 있음

# 소프트웨어 개발 표준
- 소프트웨어 개발 단계에서 품질 관리와 개발 표준을 위해 국제적으로 정하는 표준
## ISO/IEC 12207 표준
- 소프트웨어 생명 주기 프로세스
- 소프트웨어와 관련된 조직과 사람, 소프트웨어 획득자, 공급자, 개발자 등의 이해관계자들이 각자의 입장에서 수행해야 할 일을 정의하고 지속적으로 개선시키기 위한 활동

|    프로세스    | 설명                                   |
| :--------: | ------------------------------------ |
| 기본 공정 프로세스 | 획득, 공급, 개발, 운영, 유지보수 프로세스            |
| 지원 공정 프로세스 | 문서화, 형상 관리, 품질 보증, 검증, 확인, 합동 검토, 감사 |
| 조직 공정 프로세스 | 관리, 기반 구조, 개선, 교육 훈련                 |
## SPICE (ISO 15504)
- 6단계
- ==소프트웨어 처리 개선 및 능력 평가 기준== (Software Process Improvement and Capability Determination)
- 소프트웨어 프로세스에 대한 개선 및 능력 측정 기준에 대한 국제 표준

| 기준    | 설명                          |
| ----- | --------------------------- |
| 불완전   | 프로세스가 구현되지 않거나 목적 달성 불가능    |
| 수행    | 목적은 달성하지만, 계획/축적 불가능        |
| 관리    | 프로세스 수행이 계획되고 관리됨           |
| 확립    | 표준 프로세스를 이용하여 계획/관리         |
| 예측 가능 | 표준 프로세스 능력에 대해 정량적 이해/성능 예측 |
| 최적(화) | 정의된 프로세스와 표준 프로세스가 지속적으로 개선 |

## CMMI (Capability Maturity Model Integration)

| 연속적 모델 | 조직의 비즈니스 목적을 충족시키고, 위험 요소를 완화하는 데 중요한 개선 사항의 순서를 정하여 적용 가능<br>능력 레벨을 이용해 프로세스 영역별로 성숙도 평가가 가능한 모델 |

# 테일러링(Tailoring)

- 프로젝트 상황에 맞게 이미 정의된 개발 방법론의 절차, 기법, 산출물 등을 재활용하여 계획을 수립하는 기법

# 세션 (Session)
- 사용자가 웹 서버에 접속하면, 일정 시간 동안 서버와의 연결 정보/상태를 유지하며 서버와 클라이언트 간의 정보를 논리적으로 유지하는 기술
- 주로 인증 상태 유지, 사용자 활동 추적, 쇼핑몰 장바구니 기능 등에 활용
# 세션 하이재킹 (Session Higacking)
- 서버에 접속하고 있는 클라이언트들의 세션 정보를 가로채는 공격 방법
- 정상 연결을 Reset 패킷을 통해 종료시킨 후 재연결 시 침입자에게 연결
## 세션 하이재킹 탐지 방법
1. 비동기화 상태 탐지
2. ACK Storm 탐지
3. 패킷 유실 탐지 및 재전송 증가 탐지
4. 예상치 못한 접속의 리셋 탐지

## 네트워크 연결 방식

| 방식                                | 설명                                                                                                                    |
| --------------------------------- | --------------------------------------------------------------------------------------------------------------------- |
| DAS<br>(Direct Attached Storage)  | 서버와 저장 장치를 전용 케이블로 ==직접 연결== (외장 하드)<br>속도 빠르고 설치 및 운영이 간편<br>구축 및 유지보수 비용 저렴<br>다른 서버에서 접근 및 파일 공유 불가능 (확장성/유연성 떨어짐) |

## 네트워크 관련 신기술
### Wi-SUN (Wireless Smart Utility Network)
- 스마트 그리드(Smart Grid)와 연계하여 전기, 수도, 가스 등의 공급자와 사용자가 무선 네트워크를 이용해 효율적으로 관리할 수 있도록 활용하는 IEEE 802.15.4 표준 기반의 무선 통신 기술
- 저가격 및 저전력(8mA)
- 통신사 제공 서비스가 아닌 자체 자가망 구축 형태의 비면허대역(900MHz) 사용
### NFC (Near Field Communication)
- 13.56MHz 주파수 사용
- 424Kbps 속도로 데이터를 전송하는 RFID 확장 기술
- 10cm 이내에서 저전력, 비접촉식 무선 통신 기술
- 고주파(HF)를 이용하는 ISO/IEC 18092 표준으로 아주 가까운 거리에서 양방향 통신 지원
### 블루투스 (Bluetooth)
- 2.4GHz ISM 주파수 대역을 이용해 10m 이내의 근거리 디바이스 간 통신을 지원하기 위한 무선 접속 규격 (IEEE 802.15.1)
#### 블루투스 네트워크 구성

| 구성                   | 설명                                                             |
| -------------------- | -------------------------------------------------------------- |
| 피코넷<br>(Piconet)     | Master-Slave 방식<br>링크를 설정하고, 한 대의 마스터로 7대까지의 슬레이브를 연결해 네트워크 구성 |
| 스캐터넷<br>(Scatternet) | 피코넷이 여러 개 모여서 계층적이고 규모가 큰 네트워크 구성 가능                           |
### BLE (Bluetooth Low Energy)
- 저전력 기반 기기 간 근거리 무선 통신 기능을 제공하는 기술 및 규격
- 짧은 전송 거리를 극복하고, 1Mbps의 전송 속도로 2.4GHz 주파수를 사용하는 저비용으로 구성 가능한 블루투스 기술
### Zing
- 기기를 키오스크에 갖다 대면 원하는 데이터를 바로 가져올 수 있는 기술
- 10cm 이내 근접 거리에서 기가급 속도로 데이터 전송이 가능한 초고속 근접 무선 통신 기술
- 3Gbps 급의 기기 간 최대 전송 속도(무선 구간) 및 직관적 사용자 인터페이스(Touch & Get 방식) 기반의 대용량 데이터 순간 전송 제공
### BcN (Broadband covergence Network)
- 통신/방송/인터넷이 융합된 품질 보장형 광대역 멀티미디어 서비스를 언제 어디서나 끊김 없이 안전하게 이용할 수 있는 광대역 통합망
### C-V2X (Cellular-Vehicle to Everything)
- 차량이 유/무선망을 다른 차량 및 도로 등 인프라가 구축된 사물과 정보를 교환할 수 있는 자율주행자동차를 위한 통신 기술
### 애드 훅 네트워크 (Ad-hoc Network)
- 고정된 기반 망의 도움 없이 이동 노드 간에 자율적으로 구성되는 망
- 네트워크에 자율성과 융통성을 부여한 네트워크
- Peer-to-Peer 통신, 다중 홉, 이동 노드 간 동적 네트워크 구성 가능
- 망을 구성한 후 단기간 사용되는 경우나 유선망을 구성하기 어려운 경우에 적합
### UWB (Ultra Wide Band)
- 매우 낮은 전력을 사용
- 초 광대역 주파수 대역으로 디지털 데이터를 전송하는 무선 전송 기술
- 무선 디지털 펄스
- 0.5m/W 정도의 저전력으로 많은 양의 데이터를 1km 거리까지 전송 가능
### SDN (Software Defined Network)
- 소프트웨어 정의 네트워크
- 네트워크를 Control Plane(제어부), Data Plane(데이터 전달부)로 분리하여 네트워크 관리자가 보다 효율적으로 네트워크를 제어, 관리할 수 있는 기술
- 기존의 라우터, 스위치 등과 같이 하드웨어에 의존하는 네트워크 체계에서 안정성, 속도, 보안 등을 소프트웨어로 제어, 관리하기 위해 개발
- 네트워크의 세부 구성 정보에 얽매이지 않고 요구사항에 따라 네트워크를 제어 및 관리 가능
- 네트워크 장비의 펌웨어 업그레이드를 통해 사용자의 직접적인 데이터 전송 경로 관리 가능
- 기존 네트워크에는 영향을 주지 않으면서 특정 서비스의 전송 경로 수정을 통해 인터넷상에서 발생하는 문제 처리 가능
### NFV (Network Function Virtualization)
- 네트워크 기능 가상화
- 범용 하드웨어(서버/스토리지/스위치)에 가상화 기술을 적용하여 네트워크 기능을 가상 기능으로 모듈화하여 필요한 곳에 제공(스위치, 라우터 등)하는 기술
### 네트워크 슬라이싱 (Network Slicing)
- 하나의 물리적 코어 네트워크를 독립된 다수 가상 네트워크로 분리한 뒤 고객 맞춤형 서비스를 제공하는 5G 핵심 기술
- SDN과 NFV 기술을 활용해 하나의 물리적인 망에 여러 개의 논리적인 망을 만들어 비용 절감 가능
### MEC (Mobile Edge Computing/Cloud)
- 무선 기지국에 분산 클라우드 컴퓨팅 기술을 적용하여 서비스와 캐싱 콘텐츠를 이용자 단말에 가까이 전개함으로써 모바일 코어 망의 혼잡을 완화하는 기술
### 사물 인터넷 (IoT : Internet of Things)
- 각종 사물에 센서와 통신 기능을 내장하여 무선 통신을 통해 각종 사물을 인터넷에 연결하는 기술
### CoAP (Constrained Application Protocol)
- M2M 노드들 사이에서 이벤트에 대한 송수신을 비동기적으로 전송하는 REST 기반의 프로토콜
- 제약이 있는 장치들을 위한 특수한 인터넷 애플리케이션 프로토콜
### 지그비 (Zigbee)
- 근거리 통신을 지원하는 IEEE 802.15.4 표준 중 하나
- 868/915MHz, 2.4GHz 주파수 대역을 이용하는 저전력, 저속, 저비용의 근거리 무선 통신 기술
### AllJoyn
- 사물인터넷(IoT) 디바이스 간의 상호 작용을 위한 퀄컴이 개발한 오픈소스 소프트웨어 프레임워크
- 디바이스들이 서로 통신하고 협업 가능
- 이를 통해 서로 다른 제조업체의 기기들이 함께 작동할 수 있게 연결 가능
## 네트워크 장비 유형
### 대표적인 네트워크 장비
1. 브라우터 (Brouter)
	- 브리지와 라우터의 기능을 모두 갖추고 있는 네트워크 장비
2. 리피터 (Repeater)
	- 전송되는 신호가 전송 선로의 특성 및 외부 충격 등의 요인으로 인해 원래의 형태와 다르게 왜곡되거나 약해질 경우 원래의 신호 형태로 재생하여 다시 전송하는 역할 수행
3. 브리지 (Bridge)
	- LAN과 LAN을 연결하거나 LAN 안에서 컴퓨터 그룹을 연결하는 기능 수행
	- 데이터 링크 계층 중 MAC 계층에서 사용
4. 스위치 (Switch)
	- LAN과 LAN을 연결하여 훨씬 더 큰 LAN을 만드는 장치
	- OSI 7계층의 2계층에서 사용
5. 라우터 (Router)
	- LAN과 LAN의 연결 기능에 데이터 전송의 최적 경로를 선택할 수 있는 기능이 추가된 것
	- 서로 다른 LAN이나 LAN과 WAN의 연결도 수행
	- OSI 7계층의 네트워크 계층에서 동작
### 광전송 장비
- 네트워크의 스위칭 노드를 묶어 주는 시스템
- 광케이블을 이용해 비교적 긴 거리의 데이터 전송에 이용

| ----- | ------------------------------------------------------------------------------------------------------------------------------------------------------- |
| SONET | (Synchronous Optical Network)<br>고속 디지털 통신을 위한 광전송 시스템 표준 규격<br>광전송용 동기식 다중화 방식에 의한 디지털신호계위 북미 표준                                                       |
| SDH   | (Synchronous Digital Hierachy)<br>북미 표준인 SONET을 기초로 동기식 디지털 다중화 신호 계위에 관한 ITU 국제 표준 규격<br>자체 복구(Self-Healing) 기능과 SDH 프레임 내에 충분한 오버헤드 확보가 가능한 기술        |
| WDM   | (Wavelength Division Multiplexing)<br>파장이 서로 다른 복수의 광신호를 동시에 이용하는 것<br>광섬유를 다중화하는 방식<br>광섬유를 이용한 통신 기술 중 하나<br>빛의 파장 축과 파장이 다른 광선은 서로 간섭을 일으키지 않는 성질 이용 |
| DWDM  | (Dense Wavelength Division Multiplexing)<br>대용량 데이터 전송을 위해 파장 대역 채널을 조밀하게 나누어 규격화한 광전송 기술<br>일정 파장 대역에 걸쳐 수십, 수백 개 파장의 광신호를 동시에 변조                      |
| CET   | (Carrier Ethernet Transport)<br>광역 통신망에서 고속으로 데이터를 전달하고 교환하는 차세대 패킷 전송 기술<br>PTN : Packet Transport Network                                             |
### 다중화기 (Multiplexer)
- 하나의 회선을 통해 일정한 시간이나 주파수로 나누어서 전송하게 하는 장비

| 종류         | 설명                                                                                                 |
| ---------- | -------------------------------------------------------------------------------------------------- |
| 주파수 분할 다중화 | FDM : Frequency Division Multiplexing<br>하나의 주파수 대역폭을 다수의 작은 대역폭으로 분할하여 전송                         |
| 시간 분할 다중화  | TDM : Time Division Multiplexing<br>회선의 대역폭을 일정 시간으로 분할하여 전송                                       |
| 코드 분할 다중화  | CDM : Code Division Multiplexing<br>정해진 주파수 대역에 다수의 사용자가 서로 다른 코드 사용<br>→ 동일한 주파수로 동시에 다수가 접속해서 전송 |
# 로그 분석
- 시스템 내에 존재하는 공격 흔적, 피해 상황 등의 이상 현상을 로그를 통해 면밀히 분석하는 과정
- 시스템에 이상 징후가 발생했을 때 관리자가 가장 먼저 보는 것 = 로그 파일
- 시스템이 해킹을 당했다고 느끼는 경우 해킹 흔적 등을 확인하기 위해 우선적으로 로그 파일을 분석
- 어디서(IP Address), 누가(ID), 어떻게(Port) 들어와서 어떤 작업을 했는 지를 확인하려 할 때 로그 파일을 분석
## 로그 저장 경로 (Directory)
- 대부분의 로그 파일 저장 경로 = “/var/log”
- 일반적으로 텍스트 형식으로 저장

| 경로       | 설명                                |
| -------- | --------------------------------- |
| /usr/adm | 초기 유닉스, BSD 계열 → HP 9버전 SunOS 4.X |
| /var/adm | 최근 유닉스 계열 → HP 10버전 이후, Solaris   |
| /var/log | 일부 BSD 계열, 리눅스 계열                 |
| /var/run | 일부 리눅스 계열                         |
## 로그 파일 유형
- 보안 도구 등을 설치하고 나면 로그 파일들이 새로 생성

| 로그명        | 설명                                                                                  |
| ---------- | ----------------------------------------------------------------------------------- |
| wtmp/wtmpx | 사용자 로그인/로그아웃 정보<br>시스템 shutdown/reboot 정보<br>명령어 : last                             |
| utmp/utmpx | 현재 시스템에 로그인한 사용자 정보<br>명령어 : who, w users, finger                                   |
| btmp/btmpx | 로그인에 실패한 정보<br>명령어 : lastb                                                          |
| lastlog    | 사용자별 최근 로그인 시간 및 접근한 소스 호스트에 대한 정보<br>명령어 : lastlog                                 |
| sulog      | su(Switch User) 명령어 실행 성공/실패 결과에 대한 정보<br>명령어 : 텍스트 파일이라서 별도 명령어 없음                 |
| acct/pacct | 사용자별로 실행되는 모든 명령어에 대한 로그<br>명령어 : lastcomm, acctcom                                 |
| xferlog    | FTP 서비스 데이터 전송 기록 로그<br>명령어 : 텍스트 파일이라서 별도 명령어 없음                                   |
| messages   | 부트 메시지 등 시스템의 가장 기본적인 시스템 로그 파일<br>운영에 대한 전반적인 메시지를 저장<br>명령어 : 텍스트 파일이라서 별도 명령어 없음 |
| secure     | 보안과 관련된 주요 로그를 기록<br>사용자 인증과 관련된 로그를 기록<br>명령어 : 텍스트 파일이라서 별도 명령어 없음                |

## Honeypot
- 1990년대 David Clock이 처음 제안
- 비정상적인 접근을 탐지하기 위해 의도적으로 설치해 둔 시스템
- 침입자를 속여 실제 공격을 당하는 것처럼 보여줌으로써 크래커를 추적 및 공격 기법의 정보를 수집하는 역할
- 쉽게 공격자에게 노출되어야 하며 쉽게 공격이 가능한 것처럼 취약해 보여야 함
# 네트워크 보안 솔루션
- 네트워크 보안 장비는 접근 통제, 침입 차단 및 탐지, DDoS 탐지 등을 수행하는 일체형 장비들 포함
## 방화벽 (Firewall)
- 기업 내부, 외부 간 트래픽을 모니터링하여 시스템의 접근을 허용하거나 차단하는 시스템
- iptables라는 리눅스 커널에서 동작하는 방화벽
- 패킷 필터링 및 NAT 기능 제공
### 침입 차단 시스템(방화벽) 구축 유형
![](https://velog.velcdn.com/images/kik328288/post/7e6593e9-8f3e-4d7b-9f13-5a9701b5cbef/image.png)

## 웹 방화벽
- WAF : Web Application Firewall
- 일반적인 네트워크 방화벽과는 달리 웹 애플리케이션 보안에 특화된 보안 장비
- SQL 인젝션, XSS 등과 같은 웹 공격을 탐지하고 차단하는 기능
## 네트워크 접근 제어
- NAC : Network Access Control
- 단말기가 내부 네트워크에 접속을 시도할 때 이를 제어하고 통제하는 기능을 제공하는 솔루션
- 바이러스나 웜 등의 보안 위협으로부터 네트워크 제어 및 통제 기능 수행
## 침입 탐지 시스템
- IDS : Instrusion Detection System
- 시스템의 비정상 사용/오용/남용 등을 실시간 탐지
### 탐지 시스템 종류

| 종류    | 설명                                                                                                |
| ----- | ------------------------------------------------------------------------------------------------- |
| 오용 탐지 | 사전에 정립되어 입력해 둔 공격 패턴 감지 시 통보<br>이미 발견되고 정립된 공격 패턴을 입력해 두었다가 탐지 및 차단                               |
| 이상 탐지 | 평균적인 상태 기준에서 비정상적인 행위 또는 자원의 사용 감지 시 통보<br>행위 또는 통계치 기반으로 탐지하는 기법<br>알려지지 않은 공격 탐지는 가능하지만 오답률이 높음 |
### IDS 종류

|           종류            | 설명                                                                                        |
| :---------------------: | ----------------------------------------------------------------------------------------- |
|  HIDS<br>(Host-based)   | 시스템 내부 감시/분석, 내부 변화를 실시간으로 감시<br>→ 외부 침입자 작업 기록 및 추적<br>EX) OSSEC, md5deep, AIDE, Samhain |
| NIDS<br>(Network-based) | 네트워크에 접근하는 공격을 탐지하는 시스템<br>외부 침입 감시/분석, 네트워크 트래픽 감시<br>EX) Snort, Zeek                    |
- IDS 설치 가능 위치 : 패킷 라우터 들어오기 전 / 라우터 뒤 / 방화벽 뒤 / DMZ
	- DMZ : 외부 인터넷에 서비스를 제공하는 서버가 위치하는 네트워크
### 침입 탐지 시스템의 탐지 유형

| 유형  | 설명                                                                                                                                  |
| --- | ----------------------------------------------------------------------------------------------------------------------------------- |
| TP  | True Positive<br>공격자가 실제로 시스템에 침입하였고, 침입 탐지 시스템은 이를 침입으로 정확히 탐지한 탐지 유형<br>공격이 발생했으며, IDS가 이를 경고하여 정상적으로 대응이 가능한 상황                  |
| FP  | False Positive<br>정상적인 행위를 침입 탐지 시스템이 침입으로 잘못 인식하여 경고를 발생시킨 탐지 유형<br>오탐으로 인해 불필요한 경고가 생성되고, 운영자에게 혼선을 줄 수 있는 상황                     |
| TN  | True Negative<br>정상적인 행위를 침입 탐지 시스템이 정상으로 인식하여 경고를 발생시키지 않은 탐지 유형<br>문제가 없는 트래픽을 잘 구분하여 시스템 자원을 낭비하지 않은 이상적인 상황                     |
| FN  | False Negative<br>공격자가 실제로 시스템에 침입하였으나 침입 탐지 시스템은 해당 공격을 정상적인 동작으로 인식하여 침입을 탐지하지 못한 탐지 유형<br>가장 위험한 유형<br>공격을 놓쳐 보안 사고로 이어질 수 있는 상황 |

## 침입 방지 시스템
- IPS : Intrusion Prevention System
- 네트워크에 대한 공격이나 침입을 실시간으로 차단하고, 유해 트래픽에 대한 조치를 능동적으로 처리하는 시스템
## 무선 침입 방지 시스템
- WIPS : Wireless Intrusion Prevention System
- 인가되지 않은 무선 단말기의 접속을 자동으로 탐지 및 차단하고 보안에 취약한 무선 공유기를 탐지하는 시스템
- 외부 공격에 대한 내부 시스템의 보호를 위해 무선랜 환경에서의 보안 위협을 탐지
- 802.1x 기반의 사용자 인증(WEP, WPA/WPA2) 및 암호화(AES) 기술 적용

| 종류   | 설명                                                                                                                    |
| ---- | --------------------------------------------------------------------------------------------------------------------- |
| WEP  | Wired Equivalent Privacy<br>WiFi 표준에 정의되어 있는 보안 프로토콜<br>고정된 암호 키를 사용하여 보안에 취약                                         |
| TKIP | Temporal Key Integrity Protocol<br>WEP를 대체하기 위한 임시 키 무결성 프로토콜<br>패킷 당 키 할당, 키 값 재설정 등의 방식으로 암호 키 변경 가능                |
| WPA  | Wi-Fi Protected Access<br>WEP의 취약점 대안<br>RC4 암호화 프로토콜과 TKIP 프로토콜을 통해 데이터 암호화를 향상시킨 프로토콜                               |
| WPA2 | Wi-Fi Protected Access2<br>AES 암호화 알고리즘의 필수 사용과 TKIP를 대체하는 CCMP를 도입한 프로토콜<br>CCMP : 블록체인 메시지 인증 코드 프로토콜이 있는 카운터 암호 모드 |
## TCP 래퍼 (TCP Wrapper)
- 어떤 외부 컴퓨터가 접속되면 접속 인가 여부를 점검해서 인가된 경우에는 접속이 허용되고, 그 반대의 경우에는 거부할 수 있는 접근 제어 유틸리티
- 클라이언트의 IP 주소를 확인하여 시스템 관리자가 접근을 허용한 호스트들에 대해서만 서비스를 허용하기 때문에 외부의 해킹으로부터 시스템 보호 가능
## 통합 보안 시스템
- UTM : Unified Threat Management
- 방화벽, 침입 탐지 시스템, 침입 방지 시스템, VPN, 안티 바이러스, 이메일 필터링 등 다양한 보안 기능을 하나의 장비로 통합하여 제공하는 시스템
## 가상사설망
- VPN : Virtual Private Network
- 인터넷과 같은 공중망에 사설망을 구축하여 마치 전용망을 사용하는 효과를 가지는 보안 솔루션
## nmap
- 윈도우, 리눅스 포트 스캔 툴
- 서버에 열린 포트 정보를 스캐닝해서 보안 취약점을 찾는 데 사용하는 도구
# 빅데이터 (Big Data)
- 시스템, 서비스, 조직 등에서 주어진 비용, 시간 내에 처리 가능한 데이터 범위를 넘어서는 수십 페타 바이트(PB) 크기의 비정형 데이터
## 빅데이터의 특성

|          특성           | 설명                                                                                       |
| :-------------------: | ---------------------------------------------------------------------------------------- |
|  데이터의 양<br>(Volume)   | PB 수준의 대규모 데이터<br>빅데이터 분석 규모에 관련된 특성<br>디지털 정보량이 기하급수적으로 폭증하는 것을 의미                      |
| 데이터의 다양성<br>(Variety) | 정형, 비정형, 반정형의 다양한 데이터<br>빅데이터 자원 유형에 관련된 특성<br>로그, 소셜, 위치 등 데이터 유형이 다양해지는 것              |
| 데이터의 속도<br>(Velocity) | 빠르게 증가하고 수집되며, 처리되는 데이터<br>빅데이터 수집, 분석, 활용 속도와 관련된 특성<br>가치 있는 정보 활용을 위해 실시간 분석이 중요해지는 것 |
## 빅데이터 기술

### MapReduce
- 구글이 대용량 데이터를 효율적으로 처리하기 위해 개발한 분산 프로그래밍 모델
- Map 단계에서는 입력 데이터를 쪼개어 키-값(Key-Value) 쌍으로 변환
- Reduce 단계에서는 동일 키를 기준으로 데이터를 집계하여 최종 결과로 통합
- 각 노드는 메모리/디스크를 공유하지 않는 Share-Nothing 아키텍처 기반으로 독립적으로 동작
# NoSQL (Not Only SQL)
- 전통적인 RDBMS와 다른 DBMS를 지칭하기 위한 용어
- 데이터 저장에 고정된 테이블 스키마가 필요하지 않고 조인 연산 사용 불가능
- 수평적으로 확장 가능한 DBMS
## NoSQL의 특성(BASE)

| 특성                     | 설명                                                                                           |
| ---------------------- | -------------------------------------------------------------------------------------------- |
| Basically Available    | 언제든지 데이터에 접근할 수 있어야 하는 속성<br>분산 시스템이기 때문에 항상 가용성 중시                                          |
| Soft-State             | 노드의 상태는 내부에 포함된 정보에 의해 결정되는 것이 아님<br>→ 외부에서 전송된 정보를 통해 결정되는 속성<br>특정 시점에서는 데이터의 일관성이 보장되지 않음 |
| Eventually Consistency | 일정 시간이 지나면 데이터의 일관성이 유지되는 속성<br>일관성을 중시하고 지향                                                 |
## NoSQL 유형

| 유형                       | 설명                                                                                                                                 |
| ------------------------ | ---------------------------------------------------------------------------------------------------------------------------------- |
| Key-Value Store          | 키 기반 Get/Put/Delete 제공<br>메모리 기반에서 성능 우선 시스템 및 빅데이터 처리 가능 DB<br>Unique한 Key에 하나의 Value를 가지고 있는 형태<br>EX) Redis, DynamicDB          |

# 입력 데이터 검증 및 표현
- 입력 데이터로 인해 발생하는 문제들을 예방하기 위해 구현 단계에서 검증해야 하는 보안 점검 항목들
- 입력 데이터로 인해 발생하는 문제를 예방하기 위해서는 소프트웨어 개발의 구현 단계에서 유효성 검증 체계를 갖추고, 검증되지 않은 데이터의 입력을 처리할 수 있도록 구현해야 함
## 입력 데이터 검증 및 표현 취약점
### XSS (Cross Site Scripting)
- 검증되지 않은 외부 입력 데이터가 포함된 웹 페이지가 전송되는 경우, 사용자가 해당 웹 페이지를 열람함으로써 웹 페이지에 포함된 부적절한 스크립트가 실행되는 공격
- 대책
	- 특수문자 등록을 방지하기 위해 특수문자 필터링
	- HTML 태그 사용 금지
	- 자바스크립트로 시작하는 문자열은 모두 문자열 변환 처리
#### XSS 공격 유형

| 유형            | 설명                                                                                                          |
| ------------- | ----------------------------------------------------------------------------------------------------------- |
| Stored XSS    | 방문자들이 악성 스크립트가 포함된 페이지를 읽어 봄과 동시에 악성 스크립트가 브라우저에서 실행되면서 감염되는 기법                                             |
| Reflected XSS | 공격용 악성 URL을 생성한 후 이메일로 사용자에게 전송하면 사용자가 URL 클릭 시 즉시 공격 스크립트가 피해자로 반사되어 접속 사이트에 민감 정보를 공격자에게 전송하는 기법          |
| DOM XSS       | Document Object Model XSS<br>공격자는 DOM 기반 XSS 취약점이 있는 브라우저를 대상으로 조작된 URL을 이메일로 발송하고 피해자가 URL 클릭 시 피해를 당하는 기법 |

### 사이트 간 요청 위조 (CSRF)
- Cross-Site Request Forgery
- 사용자가 자신의 의지와는 무관하게 공격자가 의도한 행위를 특정 웹 사이트에 요청하게 하는 공격
- 대책
	- 입력 화면 폼을 작성할 때 GET 방식보다 POST 방식 사용
	- 입력 폼과 입력 처리 프로그램에서 세션별 CSRF 토큰을 사용하여 점검
	- 중요 기능의 경우 재인증을 통해 안전하게 실제 요청 여부를 확인하도록 구현

### SQL 삽입 (SQL Injection)
- 응용 프로그램의 보안 취약점을 이용해서 악의적인 SQL 구문을 삽입, 실행시켜서 데이터베이스의 접근을 통해 정보를 탈취하거나 조작 등의 행위를 하는 공격 기법
- 대책
	- 바인딩 매개변수 방식을 적용하여 사전에 변수 타입을 명시적으로 지정
	- 사용자로부터 입력될 수 있는 모든 값을 체크하여 필터링
	- Servlet Filter 기능 적용 (JAVA에서만 적용)

# 접근 제어 모델
## 벨라파듈라 모델 (BLP)
- Bell-LaPadula Confidentiality Model
- 군대의 보안 레벨처럼 정보의 기밀성에 따라 상하 관계가 구분된 정보를 보호하기 위해 사용
- 자신의 권한보다 낮은 보안 레벨 권한을 가진 경우 높은 보안 레벨의 문서 읽는 것 불가능하고 자신의 권한보다 낮은 수준의 문서만 읽는 것 가능
- 자신의 권한보다 높은 보안 레벨의 문서에는 쓰기가 가능하지만 보안 레벨이 낮은 문서의 쓰기 권한은 제한
# 경로 제어 프로토콜 (Routing Protocol)

| 프로토콜                                        | 설명                                                                                                                                    |
| ------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------- |
| RIP<br>(Routing Information Protocol)       | 거리 벡터 라우팅 프로토콜 / 최대 흡수 15<br>최단 경로 탐색으로 Bellman-Ford 알고리즘 사용<br>EGP 보다는 IGP에 해당하며 소규모 네트워크 환경에 적합                                     |
| OSPF<br>(Open Shortest Path First Protocol) | RIP 단점 개선 목적 / 대규모 네트워크에 널리 사용<br>최단 경로 라우팅 지원 (실시간 노드 간 거리, 링크 상태 반영)<br>멀티캐스팅 지원<br>네트워크 변화에 신속하게 대처 가능<br>다익스트라 (Dijkstra) 알고리즘 사용 |

## OWASP
- The Open Web Application Security Project
- 오픈 소스 웹 애플리케이션 보안 프로젝트
- 주로 웹을 통한 정보 유출, 악성 파일 및 스크립트, 보안 취약점 등을 연구하는 기관
# 프로젝트 관리
- 주어진 기간 내에 최소한의 비용으로 사용자를 만족시키는 시스템을 개발하기 위한 전반적인 활동
- 소프트웨어 개발 계획을 세우고 분석, 설계, 구현 등의 작업을 통제하는 것
- 소프트웨어 생명 주기의 전 과정에 걸쳐서 진행
- 소프트웨어 프로젝트를 성공적으로 수행하기 위해서는 수행할 작업의 범위, 필요한 자원, 수행 업무, 비용, 추진 일정 관리 필요
## 프로젝트 관리 대상

| 대상    | 설명                                                                                                                                  |
| ----- | ----------------------------------------------------------------------------------------------------------------------------------- |
| 계획 관리 | 프로젝트 계획, 비용 산정, 일정 계획, 조작 계획에 대한 관리                                                                                                 |
| 품질 관리 | 품질 통제 및 품질 보증                                                                                                                       |
| 범위 관리 | 이해 관계자가 요청한 모든 요구사항이 프로젝트 범위에 포함되었는지 보장하고, 필요한 작업만 수행될 수 있도록 관리<br>EX) 범위 관리 계획 수립, 요구 사항 수집, 범위 정의, 작업 분류 체계(WBS) 작성, 범위 확인, 범위 통제 |
| 위험 관리 | 위험 식별, 위험 분석 및 평가, 위험 관리 계획, 위험 감시 및 조치                                                                                             |
### 프로젝트 관리를 위한 3대 요소

| 요소                | 설명                                       |
| ----------------- | ---------------------------------------- |
| 사람                | 프로젝트 관리에서 가장 기본이 되는 인적 자원 (People)       |
| 문제                | 사용자의 입장에서 문제를 분석하여 인식 (Problem)          |
| 프로세스<br>(Process) | 소프트웨어 개발에 필요한 전체적인 작업 계획 및 구조(Framework) |

## 프로젝트 일정 관리
- 프로젝트가 일정 기한 내에 적절하게 완료될 수 있도록 관리하는 모델

| 방법                                                | 설명                                                                                                           |
| ------------------------------------------------- | ------------------------------------------------------------------------------------------------------------ |
| PERT<br>(Program Evaluation and Review Technique) | 프로젝트 작업 상호 관계를 네트워크로 표현<br>원 노드(작업)와 간선(화살표)으로 구성<br>간선에는 각 작업별 낙관치/기대치/비관치를 기재                              |
| 주 공정법(CPM)<br>(Critical Path Method)              | 노드 (작업) / 간선 (작업 후 의존 관계) / 박스 (이정표) 구성<br>간선 (화살표)의 흐름에 따라 작업 진행                                            |
| 간트차트                                              | 각 작업의 시작/종료 일정을 막대 도표를 이용하여 표현<br>시간선(Time-line) 차트 (수평 막대 길이 = 작업 시간)<br>작업 경로는 표현 불가능<br>계획 변화에 대한 적응성이 낮음 |
| 중요 연쇄 프로젝트 관리<br>(CCPM)                           | Critical Chain Project Management<br>주 공정 연쇄법<br>자원 제약 사항을 고려하여 일정을 작성하는 기법                                  |
### CPM (Critical Path Method)
- 프로젝트 내에서 각 작업이 수행되는 시간과 각 작업 사이의 관계 파악 가능
- 효과적인 프로젝트의 통제 가능
- 경영층의 과학적인 의사 결정 지원
### 간트 차트 작성 시 고려 사항
1. 작업의 순서 : 작업의 선후 관계를 명확히 해야 함
2. 작업의 기간 : 작업이 완료되기까지 소요되는 시간을 산정해야 한다.
3. 작업의 종속성 : 작업 간의 선후 관계를 파악해야 함
4. 작업의 진행 현황 : 작업의 진행 현황을 파악하고, 필요에 따라 일정 조정 필요
# 위험 관리
- 프로젝트에 내재된 위험 요소를 인식하고 그 영향을 분석하여 이를 관리하는 활동
- 프로젝트를 성공시키기 위하여 위험 요소를 사전에 예측, 대비하는 모든 기술과 활동 포함
## 위험 종류

| 종류         | 설명                                     |
| ---------- | -------------------------------------- |
| 알려진 위험     | 프로젝트 계획서, 기술적 환경, 정보 등에 의해 발견될 수 있는 위험 |
| 예측 가능한 위험  | 과거 경험으로부터 예측할 수 있는 위험                  |
| 예측 불가능한 위험 | 사전 예측이 매우 어려운 위험                       |
## 위험 관리 절차

| 절차         | 설명                                                                                         |
| ---------- | ------------------------------------------------------------------------------------------ |
| 위험 식별      | 알려지거나 예측 가능한 위험 요소를 파악하는 작업<br>위험 항목 점검 목록을 작성하여 활용                                        |
| 위험 분석 및 평가 | 프로젝트에 내재한 위험 요소를 인식하고 그 영향을 분석하는 활동<br>위험 추산 작업을 통해 수행<br>가능한 모든 위험 요소와 영향을 분석하여 의사 결정에 반영 |
| 위험 관리 계획   | 위험을 예방하고 위험 발생 시 대책을 준비하여 문서화하는 작업                                                         |
| 위험 감시 및 조치 | 위험 내용을 항상 감시하고 실제 발생 조치                                                                    |
### 위험 감시 및 조치

| 내용              | 설몀                                               |
| --------------- | ------------------------------------------------ |
| 위험 회피           | 위험 관리에 대한 최상의 전략<br>위험이 발생 될 것을 예상하고 회피하는 활동     |
| 위험 감시           | 위험 요소 징후들에 대해 지속적으로 인지하고 파악하는 활동                 |
| 위험 관리 /비상 계획 수립 | 위험 회피 전략이 실패할 경우 위험을 관리<br>위험 대비책과 비상 계획을 세우는 활동 |

# 블록체인 (Block Chain)
- 분산 데이터베이스의 한 형태
- 분산 노드 운영자에 의한 임의 조작이 불가능하도록 고안되어 지속적으로 성장하는 데이터 기록 리스트인 블록을 연결한 모음
## 블록체인 특징
1. 블록체인 기술을 화폐(비트코인) 영역만 국한할 것이 아니라 각종 금융 상품, 공증, 스토리지, 소셜 서비스 등 다양한 서비스로 확장하고자 하는 사상
2. 디지털 통화의 발행/유통/거래가 주 기능이었던 기존의 블록체인 1.0은 기존 비트코인의 한계를 극복하고 다양한 영역으로의 확장을 목표로 블록체인 2.0으로 진화
## 블록체인 기술 요소

| 요소       | 설명                                                                                                                 |
| -------- | ------------------------------------------------------------------------------------------------------------------ |
| 분산원장 기술  | DLT : Distributed Ledger Technology<br>블록체인 네트워크 참여자들이 동일한 거래 원장 데이터를 공유 및 동기화<br>중앙 기관 없이도 데이터의 신뢰성과 투명성을 확보하는 기술 |
| 스마트 계약   | Smart Contract<br>블록체인 위에서 사전에 정해진 조건이 충족되면 코드가 자동 실행<br>→ 중개자 없이 신뢰 기반으로 계약이 이행                                   |
| 합의 알고리즘  | Consensus Algorithm<br>블록체인 네트워크 내 여러 노드가 거래의 유효성과 블록의 정당성에 대해 일치된 결정을 내리는 알고리즘                                    |
| P2P 네트워크 | Peer-to-Peer Network<br>중앙 서버 없이 참여자 간 직접 연결된 네트워크 구조<br>탈중앙화를 실현하고 단일 장애 지점을 제거한 네트워크 구조                          |
## BaaS (Blockchain-as-a-Service)
- 블록체인 개발 환경을 클라우드로 서비스하는 개념
- 블록체인 네트워크에 노드의 추가 및 제거가 용이
- 블록체인의 기본 인프라를 추사화하여 블록체인 응용 프로그램을 만들 수 있는 클라우드 컴퓨팅 플랫폼
# 클라우드 컴퓨팅 (Cloud Computing)
- 인터넷을 통해 가상화된 컴퓨터 시스템 리소스 제공
- 정보를 자신의 컴퓨터가 아닌 클라우드에 연결된 다른 컴퓨터로 처리하는 기술
- 구성 가능한 컴퓨팅 자원(컴퓨터 네트워크, 데이터베이스, 서버, 스토리지, 애플리케이션, 서비스)에 대해 어디서나 접근 가능
## 클라우드 컴퓨팅 분류

| 분류                           | 설명                                                                                                                                                  |
| ---------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| 사설 클라우드<br>(Private Cloud)   | 기업 또는 조직 내부에서 보유하고 있는 컴퓨팅 자원(IDC, 서버 등)을 사용해 내부에 구축되어 운영되는 클라우드<br>자체 컴퓨팅 자원으로 모든 하드웨어, 소프트웨어, 데이터를 수용<br>직접적인 통제가 가능하며 보안성 향상 가능                   |
| 공용 클라우드<br>(Public Cloud)    | 클라우드 서비스 제공 업체에서 다중 사용자를 위한 컴퓨팅 자원 서비스를 제공하는 클라우드<br>일정한 비용을 지불하고 하드웨어, 소프트웨어 등을 사용<br>확장성, 유연성 등이 뛰어남                                              |
| 하이브리드 클라우드<br>(Hybrid Cloud) | 기업 또는 조직 내부 자원을 이용한 사설 클라우드와 공용 클라우드를 모두 사용하는 클라우드<br>사설 클라우드의 약점인 구축 비용 문제와 공용 클라우드의 약점인 보안성 확보 문제를 해결<br>사용 업무 중요도, 보안성 확보와 중요도 등에 따라 이용 형태 변경 가능 |
## 클라우드 컴퓨팅 유형

| 유형         | 설명                                                                                                                                                                                                                        |
| ---------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 인프라형 서비스   | IaaS : Infrastructure as a Service<br>서버, 스토리지 같은 시스템 자원을 클라우드로 제공하는 서비스<br>컴퓨팅 자원에 운영체제나 애플리케이션 등의 소프트웨어 탑재/실행<br>하위의 클라우드 인프라를 제어하거나 관리하지 않음<br>스토리지, 애플리케이션에 대해서는 제어권 보유                                               |
| 플랫폼형 서비스   | PaaS : Platform as a Service<br>인프라를 생성/관리하는 복잡함 없이 애플리케이션을 개발/실행/관리할 수 있게 하는 플랫폼을 제공하는 서비스<br>SaaS의 개념을 개발 플랫폼에도 확장한 방식<br>개발을 위한 플랫폼을 구축할 필요 없이, 필요한 개발 요소를 웹에서 빌려 쓸 수 있게 하는 모델<br>OS, 애플리케이션과 애플리케이션 호스팅 환경 구성의 제어권 보유 |
| 소프트웨어형 서비스 | SaaS : Software as a Service<br>소프트웨어 및 관련 데이터는 중앙에 호스팅되고 사용자는 웹 브라우저 등의 클라이언트를 통해 접속하여 소프트웨어를 서비스 형태로 이용하는 서비스<br>주문형 소프트웨어                                                                                              |

# 추가적인 5과목 개념
## 인증 권한 검증 계정 관리 (AAA)
- Authentication(인증) Authorization(권한 부여) Accounting(계정 관리)
- 시스템의 사용자가 로그인하여 명령을 내리는 과정에 대한 시스템 동작을 인증, 권한 부여, 계정 관리로 구분
- 인증 (Authentication)
	- 인증된 사용자에게 어떤 권한을 부여할 것인지 결정하는 과정
	- 일반적으로 역할이나 그룹에 기반하여 부여
	- 시스템 접근을 허용하기 전에 사용자의 신원 검증
- 권한 부여 (Authorization)
	- 검증된 사용자에게 어떤 수준의 권한과 서비스 허용
- 계정 관리 (Accounting)
	- 사용자의 자원에 대한 사용 정보를 모아서 과금, 감사, 용량 증설, 리포팅 등 수행
## SIP (Session Initiation Protocol)
- 인터넷 상에서 통신하고자 하는 지능형 단말들이 서로를 식별하여 그 위치를 찾고, 그들 상호 간에 미디어 통신 세션을 생성하거나 삭제 또는 변경하기 위한 절차를 명시한 시그널링 프로토콜
## 비컨 (Beacon)
- 블루투스 4.0(BLE) 프로토콜 기반의 근거리 무선 통신 장치
- 최대 70m 이내의 장치들과 교신할 수 있는 차세대 스마트폰 근거리 통신 기술
- 저전력으로 모바일 결제 등을 가능하게 해주는 스마트폰 근거리 통신 기술
- NFC보다 가용거리가 길고 5~10cm 단위 구별이 가능해 정확성이 높음
## Salt
- 시스템에 저장되는 패스워드들은 Hash 또는 암호화 알고리즘의 결과값으로 저장
- 이때 암호 공격을 막기 위해 똑같은 패스워드들이 다른 암호값으로 저장되도록 추가되는 값
## BYOD (Bring Your Own Device) 환경에서 주요 보안 강화 기술
1. MDM (Mobile Device Management)
	- 기기 자체를 관리하는 방식
	- 사전 등록된 모바일 장비를 통해 원격으로 개인 사용자 기기를 등록, 관리, 추적 등을 지원하는 단말 기기 관리 업무를 처리하는 BYOD 환경에서의 주요 보안 강화 기술
2. MAM (Mobile Application Management)
	- 기기 내 특정 애플리케이션만 관리하는 방식
## 인공지능 (AI: Artificial Intelligence)
- 인간의 지적 능력을 인공적으로 구현하여 컴퓨터가 인간의 지능적인 행동과 사고를 모방할 수 있도록 하는 소프트웨어
## 휴리스틱 탐색 방법 (Heuristic Search)
- 문제를 해결하기 위해 경험적 규칙이나 추정치를 사용해 탐색 공간을 효율적으로 줄이는 방법
- 주로 최적화 문제나 인공지능 분야에서 많이 사용
- 종류 : Heuristic Function, A Algorithm, Consistent Heuristic, Greedy Search, Beam Search, Hill Climbing, Simulated Annealing, Best First
### IBN (Intent-Based Networking)
- 인공지능과 머신러닝 기술을 활용하여 네트워크 동작을 모니터링하고, 문제를 예측하고, 최적의 구성을 제안하여 네트워크 관리와 운영을 간소화하고 자동화하기 위한 접근 방식
- 네트워크 관리를 더 효율적이고 자동화된 방식으로 수행하기 위해 인공지능과 머신러닝을 활용하는 개념
- 네트워크 운영자가 의도를 기반으로 네트워크 동작을 설정하고 제어할 수 있는 방식을 제공
- 네트워크의 상태를 실시간으로 모니터링하고 예측하여 문제를 사전에 감지하고 조치 가능
- 네트워크 구성을 최적화하고 필요한 변경을 자동으로 제안하며, 네트워크 운영을 간소화하고 자동화 가능
## 기계학습(MK : Machine Learning)
- 인공지능 분야 중 하나
- 인간의 학습 능력과 같은 기능을 컴퓨터에서 실현하고자 하는 기술
- 환경과의 상호작용에 기반한 경험적인 데이터로부터 스스로 성능을 향상시키는 시스템을 연구하는 기술
## Secure OS
- 기존 운영체제(OS)에 보안 기능을 갖춘 커널 이식한 운영체제
- 보안 커널은 보안 기능을 갖춘 커널로 TCB 기반으로 참조 모니터의 개념을 구현하고 실행
- 보안 커널의 보호 대상
	- 메모리와 보조 기억 장치
	- 저장된 데이터
	- 하드웨어 장치
	- 자료 구조, 명령어
### ==Secure OS 보안 기능==
1. 사용자 식별 및 인증
2. 임의적 접근 통제(DAC), 강제적 접근 통제(MAC) 기능
3. 객체 재사용 보호 기능
4. 모든 접근에 대한 통제 기능
5. 패스워드 설정, 정책 설정 시 신뢰 경로를 통한 통신 기능
6. 모든 보안 관련 기록이 저장되고 보호되는 감사 기능
7. 참조 모니터를 통해 모든 접근 통제 제어 기능
## 인증 (Authentication)
- 사용자의 정보를 확인하고 접근 권한을 검증하는 보안 절차
- 주요 유형 : 지식 기반 / 소유 기반 / 생체 기반 / 위치 기반
## 세마포어(Semaphore)
- 공유 자원에 대한 동기화와 상호 배제를 위해 사용하는 정수형 변수
- 초기 값은 동시에 접근 가능한 자원의 수
	- 초기값이 2라는 것은 2개의 프로세스가 동시에 자원을 사용할 수 있다는 뜻
- P() 연산은 자원을 요청(감소)
	- 세마포어 값을 감소시키며, 값이 음수가 되면 프로세스를 대기 상태로 만듬
- V() 연산은 자원 반환(증가)에 해당
- 임계 구역을 하나만 허용하고 싶을 경우 보통 1로 초기화
	- 뮤텍스와 유사한 구조
- 초기값이 1인 경우 : 하나의 프로세스만 임계 구역에 진입 가능
- 초기값이 0인 경우 : 초기 상태에서 자원 사용 불가능
## VRRP (Virtual Router Redundancy Protocol)
- 라우터 이중화(Backup)를 위해 사용하는 표준화된 프로토콜(RFC 3768)
- 네트워크 장애에 대비하여 백업 라우터가 자동으로 활성화될 수 있도록 지원하는 백업 라우팅 프로토콜
- 실제 여러 대의 라우터가 존재하지만, 하나의 가상 IP 주소를 대표로 사용
- 마스터 라우터가 다운되면 백업 라우터가 자동으로 역할을 승계하여 통신 유지
- 주로 고가용성을 위한 구성에 사용
## 포맷 스트링 공격 (Format String Attack)
- C언어의 printf() 함수에서 %d, %f, %x와 같은 형식 지시자(Format Specifier)를 적절히 검사하지 않을 경우 발생하는 취약점을 이용한 공격
- 공격자는 입력 값에 형식 문자열을 조작하여 스택 메모리의 RET 등을 덮어쓰거나 임의의 메모리 값을 읽고 쓸 수 있는 해킹 기법
## Tripwire
- 파일 시스템의 무결성 검사 도구
- 호스트 기반 크래커가 침입하여 백도어를 만들어 놓거나, 설정 파일을 변경했을 때 분석하는 도구
- 시스템 초기 상태의 파일 해시 정보를 저장해두고, 주기적으로 검사하여 파일 변경 여부 탐지
- 침입 이후 탐지에 매우 유용할 뿐만 아니라 무결성 입증과 변화 관리, 정책 준수 같은 다른 여러 목적에도 사용 가능
## SATAN
- Security Administrator Tool for Analyzing Networks)
- 네트워크 기반 취약점 분석 도구
- 시스템의 네트워크 서비스 취약점을 스캔하고 분석하는 데 사용
## Klaxon
- 침입 탐지 시스템 (IDS) 중 하나
- 비정상적인 접근을 탐지하여 관리자에게 경고하는 도구
- 목적 : 실시간 탐지
## Watcher
- 일반적으로 알려진 보안 도구로는 잘 등장하지 않음
- 특정한 웹 취약점 스캐너로 알려졌지만 무결성 검증이나 호스트 기반 탐지 도구 아님
## STA (Spanning Tree Algorithm)
- 브리지와 구내 정보 통신망(LAN)으로 구성된 통신망에서 루프(폐회로)를 형성하지 않으면서 연결을 설정하는 알고리즘
